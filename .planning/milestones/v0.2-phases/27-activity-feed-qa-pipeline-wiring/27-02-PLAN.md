```typescript
// QA-04: Log retrieval audit trail for this Schriftsatz draft
try {
  const { modelName } = await getModelForTier(3);
  await logRetrieval({
    schriftsatzId: draft.id,
    queryText: message,
    retrievalBelege: assemblyResult.allBelege,
    promptVersion: "schriftsatz-v1",
    modell: modelName,
  });
} catch {
  // Non-critical: QA logging failure must not fail the pipeline
}
```

Key details:
- `draft.id` is the HelenaDraft ID -- used as `schriftsatzId` for correlation
- `message` is the original user query text -- `logRetrieval` internally calls `hashQuery()` to SHA-256 hash it before storing (QA-07 PII safety)
- `assemblyResult.allBelege` contains the `RetrievalBeleg[]` from RAG assembly
- `promptVersion: "schriftsatz-v1"` is a static version string per research recommendation
- `getModelForTier(3)` returns cached model info -- fast call, no performance concern
- Wrap in try-catch: QA logging failure MUST NOT fail the Schriftsatz pipeline
- Do NOT use fire-and-forget `.catch(() => {})` pattern here -- the try-catch is more appropriate since we `await` the result for data integrity (we want the log to actually be written, unlike notifications which are truly fire-and-forget)

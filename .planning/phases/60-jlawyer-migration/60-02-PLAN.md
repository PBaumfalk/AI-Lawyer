  <name>Task 1: ETL for Akten</name>
  <files>src/lib/jlawyer/etl-akten.ts</files>
  <action>
Create `src/lib/jlawyer/etl-akten.ts` with a `migrateAkten` function that:

1. Accepts `(client: JLawyerClient, kanzleiId: string | null)` parameters
2. Calls `client.listCases()` to fetch all J-Lawyer cases
3. For each case, calls `prisma.akte.upsert` with:
   - `where: { jlawyerId: jlCase.id }` (idempotent by J-Lawyer case ID)
   - `create`: maps J-Lawyer fields → Akte fields
   - `update`: same field mapping (re-run updates metadata)
4. Returns `Partial<JLawyerMigrationStats>` with `{ akten, errors }`

Field mapping (J-Lawyer → Akte):
- `fileNumber` → `aktenzeichen` (if empty, generate placeholder `JL-${jlCase.id}`)
- `name` → `kurzrubrum`
- `reason` → `wegen`
- `subject` → `sachgebiet` via `mapSachgebiet()` helper (see below)
- `status` → `status` via `mapAkteStatus()` helper
- `dateCreated` → `angelegt` (only on create, not update)
- `kanzleiId` passed through if provided

mapSachgebiet(subject: string): Sachgebiet — maps J-Lawyer subject string to enum:
- "Arbeitsrecht" / contains "arbeit" → ARBEITSRECHT
- "Familienrecht" / "familie" → FAMILIENRECHT
- "Verkehrsrecht" / "verkehr" → VERKEHRSRECHT
- "Mietrecht" / "miet" → MIETRECHT
- "Strafrecht" / "straf" → STRAFRECHT
- "Erbrecht" / "erb" → ERBRECHT
- "Sozialrecht" / "sozial" → SOZIALRECHT
- "Inkasso" / "inkasso" → INKASSO
- "Handelsrecht" / "handel" → HANDELSRECHT
- "Verwaltungsrecht" / "verwaltung" → VERWALTUNGSRECHT
- default → SONSTIGES

mapAkteStatus(status: string): AkteStatus:
- "OPEN" / "ACTIVE" → OFFEN
- "SUSPENDED" / "SLEEPING" → RUHEND
- "ARCHIVED" → ARCHIVIERT
- "CLOSED" → GESCHLOSSEN
- default → OFFEN

Wrap each upsert in try/catch — on error push to `errors` array and continue.
Do NOT abort migration on a single case failure.

Each upsert increments `stats.akten` counter on success.

<task type="auto">
  <name>Task 2: ETL for Kontakte and Beteiligte</name>
  <files>src/lib/jlawyer/etl-kontakte.ts</files>
  <action>
Create `src/lib/jlawyer/etl-kontakte.ts` with two exported functions:

**migrateKontakte(client: JLawyerClient): Promise&lt;Partial&lt;JLawyerMigrationStats&gt;&gt;**

1. Calls `client.listContacts()` to fetch all contacts
2. For each contact, attempts `prisma.kontakt.upsert`:
   - `where: { jlawyerId: jlContact.id }` (primary dedup key)
   - If jlawyerId not found, try dedup by email: `prisma.kontakt.findFirst({ where: { email: jlContact.email } })` — if found, update that record's jlawyerId and continue
   - `create` / `update` field mapping:
     - `type PERSON` → KontaktTyp.NATUERLICH, `type ORGANIZATION` → KontaktTyp.JURISTISCH
     - `firstName` → `vorname`, `lastName` → `nachname`
     - `company` → `firma`
     - `street` → `strasse`, `zipCode` → `plz`, `city` → `ort`
     - `country || "Deutschland"` → `land`
     - `phone` → `telefon`, `mobile` → `mobil`, `fax` → `fax`
     - `email` → `email`
     - `dateOfBirth` (parse ISO string to Date if present) → `geburtsdatum`
     - `note` → `notizen`
3. Returns `{ kontakte: N, errors: [] }`

**migrateBeteiligte(client: JLawyerClient, aktenMap: Map&lt;string, string&gt;, kontakteMap: Map&lt;string, string&gt;): Promise&lt;Partial&lt;JLawyerMigrationStats&gt;&gt;**

Parameters:
- `aktenMap`: Map of jlawyerCaseId → AI-Lawyer Akte.id (built by caller from migrateAkten result)
- `kontakteMap`: Map of jlawyerContactId → AI-Lawyer Kontakt.id (built by caller)

For each entry in aktenMap (each migrated Akte):
1. Call `client.getCaseParticipants(jlCaseId)`
2. For each participant:
   - Look up `akteId = aktenMap.get(jlCaseId)` and `kontaktId = kontakteMap.get(participant.contactId)`
   - If either missing: push to errors, continue
   - Map `participant.role` to `BeteiligterRolle` via `mapBeteiligterRolle(role)`:
     - "CLIENT" / "MANDANT" → MANDANT
     - "OPPONENT" / "GEGNER" → GEGNER
     - "OPPONENT_ATTORNEY" / "GEGNERVERTRETER" → GEGNERVERTRETER
     - "COURT" / "GERICHT" → GERICHT
     - "WITNESS" / "ZEUGE" → ZEUGE
     - "EXPERT" / "SACHVERSTAENDIGER" → SACHVERSTAENDIGER
     - default → SONSTIGER
   - Call `prisma.beteiligter.upsert`:
     - `where: { akteId_kontaktId_rolle: { akteId, kontaktId, rolle } }` (Prisma compound unique)
     - `create: { akteId, kontaktId, rolle, notizen: participant.note }`
     - `update: { notizen: participant.note }`
3. Returns `{ beteiligte: N, errors: [] }`

Wrap every DB call in try/catch; never abort entire migration on single failure.

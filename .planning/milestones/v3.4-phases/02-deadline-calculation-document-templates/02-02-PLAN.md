<objective>
Add Prisma schema enhancements for Fristen/Kalender, build the FristenRechner UI (sidebar sheet + inline), Tagesuebersicht dashboard widget, Warn-Ampel, Vorfristen reminder worker, deadline management workflows (Erledigung with mandatory reason, Fristverlaengerung), Fristenzettel PDF generation, Sonderfaelle (Oeffentliche Zustellung, Auslandszustellung), and validation rules (Keine Akte ohne Frist/WV, Auto-Wiedervorlage on document add).

The Warn-Ampel (green/yellow/red/black) provides instant visual risk assessment. The BullMQ reminder job ensures no deadline is missed. The Fristenzettel provides a printable overview for court preparation and daily planning.

    **1. Prisma Schema (prisma/schema.prisma):**
    Add `FristPrioritaet` enum: SEHR_NIEDRIG, NIEDRIG, NORMAL, HOCH, DRINGEND.
    Enhance `KalenderEintrag` model with:
    - `prioritaet FristPrioritaet @default(NORMAL)`
    - `fristArt String?` (EREIGNISFRIST | BEGINNFRIST)
    - `bundesland String?` (BundeslandCode)
    - `istNotfrist Boolean @default(false)`
    - `quittiert Boolean @default(false)`, `quittiertAm DateTime?`, `quittiertVonId String?`
    - `erledigungsgrund String?` (MANDATORY when erledigt=true -- enforced in API)
    - `vorfristen DateTime[]` (array of pre-deadline dates)
    - `halbfrist DateTime?`
    - Self-relation for Hauptfrist/Vorfrist: `hauptfristId String?`, `hauptfrist KalenderEintrag? @relation("VorfristZuHauptfrist", ...)`, `vorfristEintraege KalenderEintrag[] @relation("VorfristZuHauptfrist")`
    - `sachbearbeiterId String?` with User relation
    - `serienId String?`, `serienRegel String?` for recurring events
    - `dokumentIds String[]` for linked documents
    - `sonderfall String?` -- nullable, values: "OEFFENTLICHE_ZUSTELLUNG" | "AUSLANDSZUSTELLUNG_EU" | null. Used to apply special Fristbeginn rules.
    Add indexes on `prioritaet`, `istNotfrist`, `hauptfristId`.

    Add `FristPreset` model: id, name, fristArt, dauerWochen/Monate/Tage (Int?), istNotfrist, defaultVorfristen (Int[]), kategorie, beschreibung, sortierung, aktiv, createdAt, updatedAt.

    Run `npx prisma db push` to apply.

    **2. Fristen Rechner API (src/app/api/fristen/rechner/route.ts):**
    POST endpoint accepting: `{ zustellungsdatum, fristArt, dauer, bundesland, section193, richtung?, presetId?, sonderfall? }`.
    Validate with zod. The `sonderfall` field accepts "OEFFENTLICHE_ZUSTELLUNG" or "AUSLANDSZUSTELLUNG_EU".
    - Oeffentliche Zustellung (Section 188 ZPO): Frist begins 1 month after Aushang. Add 1 month to zustellungsdatum before calculating the regular Frist.
    - Auslandszustellung (EU-ZustVO): Extended delivery period. Add configurable extra days (default 14 days for EU, 28 days for non-EU) to account for international delivery time before Fristbeginn.
    Call `berechneFrist()` or `berechneFristRueckwaerts()` from the lib. Also call `berechneVorfristen()` and `berechneHalbfrist()`. Return full FristErgebnis with Vorfristen and Halbfrist plus sonderfall metadata.
    If `presetId` provided, load preset from DB and use its defaults (overridable).
    Auth required (any authenticated user).

    **3. Presets CRUD API (src/app/api/fristen/presets/route.ts):**
    GET: List all active presets (sorted by sortierung). Any authenticated user.
    POST: Create preset. ADMIN only.
    PUT: Update preset. ADMIN only.
    DELETE: Soft-delete (set aktiv=false). ADMIN only.
    Seed default presets on first GET if none exist (using DEFAULT_FRISTEN_PRESETS from lib).

    Extend GET to support filtering by prioritaet, istNotfrist, erledigt, and date range.
    When creating a FRIST type entry with vorfristen, also create child KalenderEintrag records (typ=FRIST, linked via hauptfristId) for each Vorfrist date.

    In the Akte creation/update API routes (src/app/api/akten/route.ts and src/app/api/akten/[id]/route.ts), add a validation check: after creating/updating an Akte, query KalenderEintrag for that Akte where erledigt=false. If NO active Frist AND no active Wiedervorlage exists, return a warning field in the response: `{ warning: "Diese Akte hat weder eine laufende Frist noch eine Wiedervorlage." }`. This is a WARNING (not a blocking error) per user decision.

    In the document upload route (src/app/api/dokumente/route.ts POST), after successfully creating a Dokument record linked to an Akte, automatically create a KalenderEintrag with: typ="WIEDERVORLAGE", prioritaet=DRINGEND, titel="Neues Dokument: {dokumentName}", datum=tomorrow (next business day), akteId=dokument.akteId, verantwortlichId=akte.verantwortlichId (inherited from Akte). Log this auto-creation in audit trail.

    GET endpoint accepting query params: `?format=daily|akte&datum=2026-03-15&akteId=xxx`.
    - Format "daily": Queries all KalenderEintrag where typ=FRIST, erledigt=false, and datum within the given day/week. Groups by urgency. Includes: Aktenzeichen, Fristtyp, Zustellung, Fristende, Vorfristen, Bundesland, Verantwortlicher.
    - Format "akte": Queries all running Fristen for a specific Akte. Same fields.
    Generate PDF using pdf-lib (install if not present): create a clean table layout with header "Fristenzettel -- [Datum/Akte]", columns for each field, FristenAmpel color as colored circle, sorted by urgency.
    Return PDF with Content-Type: application/pdf and Content-Disposition: attachment.
    Auth: any authenticated user.

For each match, call `createNotification()` (existing from Phase 1) with appropriate urgency. Handle escalation: if Vorfrist unquittiert for X hours, notify Vertreter/Admin (read escalation config from SystemSettings).
    Register repeatable job: `*/5 7-19 * * 1-5` (every 5 min, Mon-Fri 7-19h -- per Kanzlei-Arbeitszeiten, loaded from settings).
    Also check for overdue Fristen (datum < now, erledigt=false) and send alarm to Verantwortlicher + Admin.

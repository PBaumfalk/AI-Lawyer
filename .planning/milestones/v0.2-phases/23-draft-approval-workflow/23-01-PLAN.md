**3. Implement the ENTWURF gate using Prisma Client Extensions in src/lib/db.ts:**

Replace the current simple PrismaClient instantiation with a `$extends` wrapper. The pattern:

```typescript
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as {
  prisma: ReturnType<typeof createExtendedPrisma> | undefined;
};

function createExtendedPrisma() {
  const basePrisma = new PrismaClient({
    log: process.env.NODE_ENV === "development"
      ? ["query", "error", "warn"]
      : ["error"],
  });

  return basePrisma.$extends({
    query: {
      dokument: {
        async create({ args, query }) {
          // BRAK 2025 / BRAO §43: AI-created documents MUST be ENTWURF
          if (args.data.erstelltDurch === "ai") {
            args.data.status = "ENTWURF";
          }
          return query(args);
        },
        async update({ args, query }) {
          // Prevent AI-created documents from leaving ENTWURF
          // unless a human reviewer (freigegebenDurchId) is set
          if (
            args.data &&
            typeof args.data === "object" &&
            "status" in args.data &&
            args.data.status !== "ENTWURF" &&
            !("freigegebenDurchId" in args.data && args.data.freigegebenDurchId)
          ) {
            // Look up original record to check if AI-created
            const existing = await basePrisma.dokument.findUnique({
              where: args.where,
              select: { erstelltDurch: true },
            });
            if (existing?.erstelltDurch === "ai") {
              args.data.status = "ENTWURF";
            }
          }
          return query(args);
        },
      },
    },
  });
}

export const prisma = globalForPrisma.prisma ?? createExtendedPrisma();

if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma;
```

**IMPORTANT NOTES:**
- The type of `prisma` changes from `PrismaClient` to the extended type. If TypeScript complains in consuming files, add a type assertion at the export: export the value and add `export type ExtendedPrismaClient = typeof prisma;` for consumers that need the type.
- The `$extends` gate fires on ALL dokument.create/update calls across the entire application. This is the safety net.
- Inside `$transaction` callbacks, the gate may NOT fire (Prisma 5.22 behavior). The accept logic in draft-service.ts MUST explicitly set `status: "ENTWURF"` when creating Dokument records within transactions.

**1. Create `src/lib/helena/draft-service.ts` with these functions:**

**acceptDraft(draftId: string, reviewerId: string): Promise<AcceptResult>**
- Use `prisma.$transaction` to:
  - Load draft with `findUniqueOrThrow`, verify `status === "PENDING"`
  - Based on `draft.typ`:
    - `DOKUMENT`: Create `prisma.dokument.create` with `akteId`, `name: draft.titel`, `dateipfad` from `draft.meta.dateipfad` (or placeholder `"helena-draft/${draftId}"`), `mimeType` from meta or `"text/plain"`, `groesse` from meta or `inhalt.length`, `erstelltDurch: "ai"`, `status: "ENTWURF"` (explicit -- safety inside tx), `createdById: reviewerId`
    - `FRIST`: Create `prisma.kalenderEintrag.create` with `akteId`, `typ: "FRIST"` (KalenderTyp enum), `titel: draft.titel`, `beschreibung: draft.inhalt`, `datum` from `draft.meta.datum`, `verantwortlichId: reviewerId`, `ganztaegig: draft.meta.ganztaegig ?? true`
    - `NOTIZ`: Create `prisma.aktenActivity.create` with `akteId`, `typ: "NOTIZ"`, `titel: draft.titel`, `inhalt: draft.inhalt`, `userId: reviewerId`
  - Update draft: `status: "ACCEPTED"`, `reviewedById: reviewerId`, `reviewedAt: new Date()`, `undoExpiresAt: new Date(Date.now() + 5000)`
  - Create `AktenActivity` with `typ: "HELENA_DRAFT"`, `titel: "Helena-Entwurf angenommen"`, `meta: { draftId, createdId, draftTyp }`
- Return `{ draftId, createdId, typ }`

**rejectDraft(draftId, reviewerId, feedback: { categories?: string[], text?: string, noRevise?: boolean })**
- Load draft, verify PENDING
- Update draft: `status: "REJECTED"`, `feedback: JSON.stringify({ categories, text, noRevise, reviewerId, reviewedAt })`, `reviewedById`, `reviewedAt`, `feedbackCategories: feedback.categories ?? []`, `noRevise: feedback.noRevise ?? false`
- If feedback has categories or text: upsert HelenaMemory for `draft.akteId`, add to `content.rejectionPatterns[]` array (keep last 50 entries). Each pattern: `{ draftTyp, categories, text, timestamp }`
- Create AktenActivity: "Helena-Entwurf abgelehnt"
- Return `{ draftId, shouldAutoRevise: !feedback.noRevise && (feedback.categories?.length > 0 || !!feedback.text), akteId, parentDraftId: draftId, revisionCount: draft.revisionCount + 1 }`

**undoAcceptDraft(draftId: string, createdId: string, typ: string, reviewerId: string)**
- Load draft, verify `status === "ACCEPTED"` and `undoExpiresAt > new Date()`
- Delete the created record based on typ (DOKUMENT: `prisma.dokument.delete`, FRIST: `prisma.kalenderEintrag.delete`, NOTIZ: `prisma.aktenActivity.delete` where meta.draftId matches)
- Revert draft: `status: "PENDING"`, clear `reviewedById`, `reviewedAt`, `undoExpiresAt`
- Delete the "angenommen" AktenActivity

**editDraft(draftId: string, reviewerId: string, updates: { titel?: string, inhalt?: string, meta?: Record<string, unknown> })**
- Load draft, verify PENDING or EDITED
- Update draft with provided fields, set `status: "EDITED"` then immediately back to `"PENDING"` (per user decision: "After editing any type: draft returns to ausstehend for another review")
- Actually just leave status as PENDING after edit (EDITED status is for the intermediate state, but the net result is PENDING)
- Create AktenActivity: "Helena-Entwurf bearbeitet"

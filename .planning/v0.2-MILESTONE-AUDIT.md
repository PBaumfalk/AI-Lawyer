## Executive Summary

All 9 phases are **complete with passing verifications**. The Helena agent system is feature-complete at the per-phase level: ReAct loop, tool system, Schriftsatz orchestrator, task system, draft approval workflow, background scanner with alerts, per-Akte memory, activity feed UI, and QA gates are all built and individually verified.

**However, the cross-phase integration check found 3 broken E2E flows** that prevent certain features from working at runtime:
1. Inline draft review in the activity feed calls non-existent API sub-paths (404)
2. Helena drafts never appear in the activity feed (no AktenActivity created at draft creation)
3. QA retrieval log is never written by the Schriftsatz pipeline (release gate reads empty data)

These are **wiring gaps**, not missing implementations — the components exist on both sides but are not connected.

### Integration Gap Details

#### Gap 1 — CRITICAL: Inline Draft Review Calls Wrong API Paths

**File:** `src/components/akten/activity-feed-entry.tsx` lines 226, 241, 259
**Issue:** `DraftReviewActions` calls:
- `PATCH /api/helena/drafts/${draftId}/accept` (404)
- `PATCH /api/helena/drafts/${draftId}/reject` (404)
- `PATCH /api/helena/drafts/${draftId}/edit` (404)

**Correct API:** `PATCH /api/helena/drafts/${draftId}` with body `{ action: "accept"|"reject"|"edit", ...fields }`
**Impact:** Every accept/reject/edit from the activity feed silently fails. The `/entwuerfe` page (DraftInbox) and DraftPinnedSection use the correct API and work fine.
**Affected:** DRFT-02, DRFT-03, DRFT-04, UI-06
**Fix:** Change 3 fetch URLs to use the correct endpoint with action in body.

#### Gap 2 — CRITICAL: Helena Drafts Never Appear in Activity Feed

**Files:** Write tools (`create-draft-*.ts`, `create-notiz.ts`) and `schriftsatz/index.ts`
**Issue:** When Helena creates a draft via `helenaDraft.create()`, no `AktenActivity(typ="HELENA_DRAFT", meta: {draftId, draftStatus: "PENDING"})` is written. The feed entry condition `entry.meta?.draftStatus === "PENDING"` at `activity-feed-entry.tsx:165` never triggers.
**Impact:** Drafts are only visible in the DraftPinnedSection and `/entwuerfe` page, not in the chronological activity feed. Inline review is unreachable.
**Affected:** UI-05, TASK-03
**Fix:** Add `prisma.aktenActivity.create({ data: { akteId, typ: "HELENA_DRAFT", meta: { draftId, draftStatus: "PENDING", draftTyp } } })` to all 5 draft creation sites.

#### Gap 3 — MAJOR: QA Retrieval Log Never Written

**File:** `src/lib/helena/qa/retrieval-log.ts`
**Issue:** `logRetrieval()` is defined but has zero callers in production code. The Schriftsatz pipeline has `retrieval_belege` in scope after assembly but never calls `logRetrieval()`. `SchriftsatzRetrievalLog` table is always empty.
**Impact:** Release gate always reads `{ totalLogs: 0 }` and the QA dashboard always shows "KEINE DATEN". Metrics infrastructure is complete but disconnected.
**Affected:** QA-04, QA-07
**Fix:** Call `logRetrieval()` from `runSchriftsatzPipeline()` in `schriftsatz/index.ts` after successful draft creation, passing `retrieval_belege`, `promptVersion`, `modell`, and `queryHash`.

<task type="auto">
  <name>Task 1: Download API + Upload API with Mandant Ordner + Anwalt notification</name>
  <files>src/app/api/portal/akten/[id]/dokumente/[dId]/download/route.ts, src/app/api/portal/akten/[id]/dokumente/upload/route.ts</files>
  <action>
1. **Create GET /api/portal/akten/[id]/dokumente/[dId]/download** endpoint:
   - Verify Mandant session and Akte access via portal auth
   - Load the Dokument: `prisma.dokument.findFirst({ where: { id: dId, akteId, mandantSichtbar: true } })`
   - If not found or mandantSichtbar=false: return 404 `{ error: "Dokument nicht gefunden" }`
   - Generate presigned URL: `getDownloadUrl(dokument.dateipfad)` — 1hr expiry
   - Return `{ url: presignedUrl, name: dokument.name, mimeType: dokument.mimeType }`
   - Do NOT expose dateipfad directly

2. **Create POST /api/portal/akten/[id]/dokumente/upload** endpoint:
   - Verify Mandant session and Akte access via portal auth
   - Accept multipart/form-data with "file" field (single file)
   - Enforce 50MB file size limit for portal uploads (lower than internal 100MB — per Claude's discretion for portal security)
   - Auto-create "Mandant" Ordner: No separate creation needed — just set `ordner: "Mandant"` on the document
   - Upload pipeline (simplified, NO OCR, NO RAG, NO preview, NO Wiedervorlage):
     ```typescript
     const buffer = Buffer.from(await file.arrayBuffer());
     const storageKey = generateStorageKey(akteId, file.name);
     await uploadFile(storageKey, buffer, file.type, file.size);
     const dokument = await prisma.dokument.create({
       data: {
         akteId,
         name: file.name,
         dateipfad: storageKey,
         mimeType: file.type || "application/octet-stream",
         groesse: file.size,
         ordner: "Mandant",
         erstelltDurch: "mandant",
         ocrStatus: "NICHT_NOETIG",
         createdById: /* Mandant's linked internal user or use Akte's anwaltId as fallback */,
         mandantSichtbar: true, // Mandant's own uploads are visible to them by default
       },
     });
     ```
   - **createdById strategy:** Since Mandant is not an internal User (different auth model from Phase 43/44), use the Akte's `anwaltId` as createdById (the Dokument model requires a User FK). Set `erstelltDurch: "mandant"` to distinguish origin.
   - **Notify Anwalt:** After successful upload:
     ```typescript
     const akte = await prisma.akte.findUnique({ where: { id: akteId }, select: { anwaltId: true, aktenzeichen: true, kurzrubrum: true } });
     if (akte?.anwaltId) {
       await createNotification({
         userId: akte.anwaltId,
         type: "document:mandant-upload",
         title: "Mandant-Upload",
         message: `Neues Dokument "${file.name}" in Akte ${akte.aktenzeichen} hochgeladen`,
         data: { akteId, dokumentId: dokument.id, fileName: file.name },
       });
     }
     ```
   - Also create an AktenActivity:
     ```typescript
     await prisma.aktenActivity.create({
       data: {
         akteId,
         typ: "DOKUMENT",
         titel: `Mandant hat Dokument "${file.name}" hochgeladen`,
         meta: { dokumentId: dokument.id, erstelltDurch: "mandant" },
         // mandantSichtbar: true (if Phase 45 adds this field)
       },
     });
     ```
   - Return `{ dokument: { id, name, groesse, createdAt } }` with status 201
   - Use `logAuditEvent()` for upload action
  </action>

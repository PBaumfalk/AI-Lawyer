2. Define `SchriftsatzPipelineOptions` interface:

   - userRole: UserRole
   - userName: string
   - akteId: string (non-nullable â€” Schriftsatz always requires Akte context)

   - abortSignal?: AbortSignal

3. Define `SchriftsatzPipelineResult` interface:
   - status: "complete" | "needs_input" | "error"
   - schriftsatz?: Schriftsatz (the full validated schema â€” present when status=complete)
   - rueckfrage?: string (present when status=needs_input â€” the conversational question to ask the user)
   - slotState?: SlotValues (current slot state for resuming after Rueckfrage)
   - intentState?: IntentResult (current intent for resuming)
   - draftId?: string (HelenaDraft ID if draft was created)
   - warnungen: ErvWarnung[] (always present, may be empty)
   - retrieval_belege: RetrievalBeleg[] (always present, may be empty)
   - totalTokens: { prompt: number, completion: number }

4. Export `runSchriftsatzPipeline(options: SchriftsatzPipelineOptions): Promise<SchriftsatzPipelineResult>`:

   Stage 1 â€” Intent Recognition:
   - Call onStepUpdate({ stage: "intent", detail: "Analysiere Anfrage..." })
   - Build AkteContext via buildAkteContext(prisma, akteId)
   - Call recognizeIntent(message, akteContext)
   - If confidence < 0.5, return { status: "needs_input", rueckfrage: "Ich bin mir nicht sicher, was fuer einen Schriftsatz du brauchst. Kannst du genauer beschreiben...?" }

   Stage 2 â€” Slot Filling:
   - Call onStepUpdate({ stage: "slots", detail: "Sammle Informationen..." })
   - Pre-fill from Akte via prefillSlotsFromAkte(prisma, akteId, intent)
   - Merge with userSlotValues (if resuming after Rueckfrage)
   - Call fillSlots(intent, prefilled, userSlotValues ?? {})
   - If not vollstaendig: return { status: "needs_input", rueckfrage: slotResult.rueckfrage, slotState: slotResult.slots, intentState: intent }
   - The caller (helena index.ts or task processor) is responsible for collecting the user's answer and calling the pipeline again with userSlotValues updated

   Stage 3+4 â€” RAG Assembly:
   - Call onStepUpdate({ stage: "assembly", detail: "Erstelle Schriftsatz..." })
   - Get KlageartDefinition from registry
   - Call assembleSchriftsatz(intent, slotResult.slots, klageartDef)
   - This stage does the heavy lifting: RAG retrieval per section + LLM generation

   Stage 5 â€” ERV Validation:
   - Call onStepUpdate({ stage: "validation", detail: "Pruefe formale Anforderungen..." })
   - Extract unresolved placeholders from assembled schriftsatz
   - Call validateErv(assembled schriftsatz, klageartDef, slotResult.slots)
   - Set vollstaendig to false if any unresolved placeholders or KRITISCH warnings

   Stage 6 â€” Draft Creation:
   - Call onStepUpdate({ stage: "draft", detail: "Erstelle Entwurf..." })
   - Build final Schriftsatz object by merging assembly result with validation output
   - Create HelenaDraft via prisma.helenaDraft.create:
     - akteId: akteId
     - userId: userId
     - typ: "DOKUMENT"
     - status: "PENDING"
     - titel: "Schriftsatz: {klageart.label} â€” {intent.stadium}" in German
     - inhalt: Render the Schriftsatz as formatted markdown (rubrum, antraege, sachverhalt etc. concatenated with section headers)
     - meta: The full SchriftsatzSchema object as JSON (machine-readable for downstream processing)
   - Return { status: "complete", schriftsatz, draftId, warnungen, retrieval_belege, totalTokens }

   Error handling:
   - Wrap the entire pipeline in try/catch
   - On error: return { status: "error", warnungen: [{ typ: "FORM", schwere: "KRITISCH", text: error.message }], retrieval_belege: [], totalTokens: { prompt: 0, completion: 0 } }
   - Check abortSignal.aborted before each stage

5. Export `isSchriftsatzIntent(message: string): boolean`:

   - Also match: "erstelle.*klage", "verfasse.*antrag", "formuliere.*berufung", "schreibe.*schriftsatz"
   - Also match aussergerichtlich patterns: "abmahnung", "kuendigungsschreiben", "vergleichsvorschlag", "aufforderungsschreiben"
   - This is the routing check used by helena/index.ts

6. Export `renderSchriftsatzMarkdown(schriftsatz: Schriftsatz): string`:
   - Render the SchriftsatzSchema as formatted German markdown with:
     - "## Rubrum" header with gericht, parties formatted
     - "## Antraege" with numbered list
     - "## Sachverhalt" with full text
     - "## Rechtliche Wuerdigung" with full text

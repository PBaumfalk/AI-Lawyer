1. Install tsdav: `npm install tsdav`

2. Create src/lib/caldav/client.ts -- tsdav wrapper:
- createCalDavClient(provider: CalDavProvider, serverUrl: string, credentials: { username: string, password?: string, oauthTokens?: { accessToken: string } }): Promise<DAVClient>
  - For GOOGLE: use DAVClient with authMethod "Oauth" and headers { Authorization: `Bearer ${accessToken}` }
  - For APPLE: use DAVClient with authMethod "Basic" and credentials { username, password }
  - Call client.login() and return client
- fetchCalendars(client: DAVClient): Promise<CalDavCalendarInfo[]>
  - Call client.fetchCalendars() and map to CalDavCalendarInfo[]
- fetchEvents(client: DAVClient, calendarUrl: string, options?: { ctag?: string }): Promise<{ events: CalDavEvent[], ctag: string }>
  - Call client.fetchCalendarObjects({ calendar: { url: calendarUrl } })
  - Parse iCalendar VEVENT data from each object into CalDavEvent
  - Return events + calendar CTag
- createEvent(client: DAVClient, calendarUrl: string, event: CalDavEvent): Promise<{ uid: string, etag: string }>
  - Build iCalendar VCALENDAR/VEVENT string from CalDavEvent
  - Call client.createCalendarObject({ calendar: { url: calendarUrl }, filename: `${uid}.ics`, iCalString })
  - Return UID and ETag
- updateEvent(client: DAVClient, calendarUrl: string, eventUrl: string, event: CalDavEvent, etag: string): Promise<{ etag: string }>
  - Build updated iCalendar string
  - Call client.updateCalendarObject({ calendarObject: { url: eventUrl, data: iCalString, etag } })
- deleteEvent(client: DAVClient, eventUrl: string, etag: string): Promise<void>
  - Call client.deleteCalendarObject({ calendarObject: { url: eventUrl, etag } })

Use createLogger("caldav") for logging. Handle tsdav errors gracefully, wrap in try/catch returning typed errors.

3. Create src/app/api/caldav-konten/route.ts:
- GET: List all CalDavKonto for current user (auth via getServerSession). Return array with syncStatus, provider, name, letzterSync. Do NOT return passwortEnc or oauthTokens.
- POST: Create new CalDavKonto. Accept { provider, name, serverUrl, benutzername, passwort?, oauthTokens? }.
  - Encrypt passwort with encryptCalDavCredential if provided
  - Store oauthTokens as JSON if provided
  - Validate connection by calling createCalDavClient + fetchCalendars
  - If validation fails, return 400 with error message
  - If validation succeeds, store konto with syncStatus = VERBUNDEN
  - Return created konto (without secrets) + available calendars list

4. Create src/app/api/caldav-konten/[id]/route.ts:
- GET: Return single konto detail (without secrets) + available calendars
- PATCH: Update konto fields (name, selectedCalendarUrl, aktiv). If credentials change, re-validate connection.
- DELETE: Delete konto and all associated CalDavSyncMapping records (Prisma cascade handles this)

All endpoints follow existing patterns: getServerSession for auth, prisma for DB, try/catch with NextResponse.json error responses.

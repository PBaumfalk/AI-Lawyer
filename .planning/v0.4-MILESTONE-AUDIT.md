## Cross-Phase Integration

### Integration Issues

**1. Missing Quest Hook on Erledigt Route (QUEST-03)**

`/api/kalender/[id]/erledigt/route.ts` is the route called by the KalenderListe checkbox when a user marks a Frist as done. This route does NOT call `enqueueQuestCheck()`. The hook exists only on the general PATCH `/api/kalender/[id]/route.ts` (used by the edit dialog).

- **Impact:** The "Siegel des Tages" quest (complete 3 Fristen) will not fire for the primary UI interaction path
- **Fix:** Add `enqueueQuestCheck(session.user.id)` after the DB update in `/api/kalender/[id]/erledigt/route.ts`
- **Affected:** QUEST-03

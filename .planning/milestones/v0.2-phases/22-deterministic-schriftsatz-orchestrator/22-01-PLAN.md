Create the `src/lib/helena/schriftsatz/` directory and two foundational files.

**schemas.ts** — All Zod schemas for the Schriftsatz pipeline:

1. `PartySchema` — z.object with name (string), anschrift (string optional), vertreter (string optional), rolle (z.enum KLAEGER/BEKLAGTER/ANTRAGSTELLER/ANTRAGSGEGNER)

2. `AnlageSchema` — z.object with nummer (string, e.g. "K1", "B2"), bezeichnung (string), dokumentId (string optional for reference to Akte Dokument)

3. `BeweisangebotSchema` — z.object with behauptung (string), beweismittel (string), anlagenNummer (string optional, references AnlageSchema.nummer)

4. `RetrievalBelegSchema` — z.object with quelle (z.enum gesetz/urteil/muster/akte_dokument), chunkId (string), referenz (string, e.g. "BGB SS 626"), score (number), auszug (string, first ~200 chars)

5. `ErvWarnungSchema` — z.object with typ (z.enum INHALT/FORM/FRIST), schwere (z.enum INFO/WARNUNG/KRITISCH), text (string), feld (string optional)

6. `IntentResultSchema` — z.object with:
   - rechtsgebiet: z.enum matching Prisma Sachgebiet values (ARBEITSRECHT, ZIVILRECHT=mapped from SONSTIGES or add ZIVILRECHT, FAMILIENRECHT, MIETRECHT, STRAFRECHT, ERBRECHT, SOZIALRECHT, HANDELSRECHT, VERWALTUNGSRECHT, SONSTIGES)
   - klageart: z.string().describe("Specific filing type ID from registry, e.g. kschg_klage, lohnklage, ev_antrag, klageerwiderung, abmahnung, generic")
   - stadium: z.enum(ERSTINSTANZ, BERUFUNG, REVISION, BESCHWERDE, EV, AUSSERGERICHTLICH)
   - rolle: z.enum(KLAEGER, BEKLAGTER)
   - gerichtszweig: z.enum(ARBG, LG, AG, OLG, LAG, BGH, BAG, VG, SG, FG)
   - gericht: z.string().optional().describe("Specific court name if derivable")
   - confidence: z.number().min(0).max(1)
   - begruendung: z.string()

7. `SchriftsatzSchema` — z.object with:
   - Metadata: klageart (string), rechtsgebiet (string), stadium (string), rolle (string), gerichtszweig (string), gericht (string)
   - rubrum: z.object with gericht (string), aktenzeichen (string optional), klaeger (PartySchema), beklagter (PartySchema), wegen (string), streitwert (number optional)
   - antraege: z.array(z.string()) — numbered claims/motions
   - sachverhalt: z.string() — full narrative
   - rechtlicheWuerdigung: z.string() — full legal analysis with {{ERGAENZUNG}} placeholders
   - beweisangebote: z.array(BeweisangebotSchema)
   - anlagen: z.array(AnlageSchema)
   - kosten: z.object with streitwert (number optional), gerichtskosten (number optional), anwaltskosten (number optional), hinweise (z.array(z.string()))
   - formales: z.object with datum (string), unterschrift (string), hinweise (z.array(z.string()))
   - retrieval_belege: z.array(RetrievalBelegSchema)
   - unresolved_platzhalter: z.array(z.string()) — unfilled {{PLATZHALTER}} names
   - vollstaendig: z.boolean() — false if any Pflichtfelder still placeholders
   - warnungen: z.array(z.string()) — ERV validator warnings

Export all schemas and their inferred types (e.g., `export type Schriftsatz = z.infer<typeof SchriftsatzSchema>`).

**klageart-registry.ts** — Data-driven registry of filing types:

1. `SlotDefinition` interface: key (string, UPPER_SNAKE_CASE), label (string, German), type (z.enum "text"/"date"/"currency"/"number"/"boolean"), required (boolean), prefillFrom (string optional — "mandant"/"gegner"/"akte"/"gericht")

2. `SectionConfig` interface: id (string), label (string), ragSources (array of "gesetz"/"urteil"/"muster"), ragQuery (string template with {{SLOT}} placeholders), generateViaLlm (boolean)

3. `StreitwertRegel` interface: typ (string, e.g. "VIERTELJAHRESGEHALT"/"FESTBETRAG"/"MANUELL"), faktor (number optional), festbetrag (number optional)

4. `ErvPruefung` interface: id (string), check (string), params (Record<string, string>)

5. `KlageartDefinition` interface: id (string), label (string), rechtsgebiet (Sachgebiet from Prisma or string), stadien (string[]), requiredSlots (SlotDefinition[]), optionalSlots (SlotDefinition[]), sections (SectionConfig[]), streitwertRegel (StreitwertRegel), ervPruefungen (ErvPruefung[])

6. `KLAGEART_REGISTRY: Record<string, KlageartDefinition>` with at least:
   - `kschg_klage` (Kuendigungsschutzklage) — goldstandard, fully detailed: required slots (klaeger_name, beklagter_name, kuendigungsdatum, zugang_datum, eintrittsdatum, bruttogehalt, berufsbezeichnung, kuendigungsart), sections (rubrum, antraege, sachverhalt, rechtliche_wuerdigung, beweisangebote, anlagen, kosten, formales), streitwert 3x Bruttogehalt, ERV checks (kschg_3_wochen_frist, betriebsrat_anhoerung)
   - `lohnklage` (Lohn-/Gehaltsklage) — goldstandard: required slots (klaeger_name, beklagter_name, zeitraum_von, zeitraum_bis, monatsbetrag, zahlungsgrund), streitwert = ausstehende Summe
   - `ev_antrag` (Einstweilige Verfuegung/Einstweiliger Rechtsschutz): required slots (antragsteller, antragsgegner, verfuegungsanspruch, verfuegungsgrund), streitwert manual
   - `klageerwiderung` (generic): required slots (beklagter_name, klaeger_name, az, klage_datum), streitwert from Klageschrift
   - `berufung` (generic): required slots (berufungsklaeger, berufungsbeklagter, urteil_datum, urteil_az, berufungsgruende)
   - `abmahnung` (aussergerichtlich): required slots (absender, empfaenger, verstoss, frist)
   - `generic` — catch-all with minimal required slots (partei_a, partei_b, betreff), all sections enabled with generic RAG queries

7. `getKlageartDefinition(id: string): KlageartDefinition` — returns the matching definition or falls back to "generic"

Use {{UPPER_SNAKE_CASE}} convention for all placeholder keys (per amtliche Muster standard, NOT dotted.key). This is the unified Platzhalter-Standard for Schriftsatz output.

**intent-router.ts** — Stage 1 of the pipeline: Intent recognition via generateObject.

1. Import `generateObject` from "ai", `getModelForTier` from "../complexity-classifier", and `IntentResultSchema` from "./schemas".

2. Define `AkteContext` interface: akteId (string), sachgebiet (string | null), kurzrubrum (string | null), beteiligte (array of {rolle: string, name: string}), gegenstandswert (number | null), dokumente (array of {name: string, tags: string[]})

3. Define `INTENT_ROUTER_SYSTEM_PROMPT` as a German system prompt instructing the model to:
   - Identify the Rechtsgebiet, Klageart, Stadium, Rolle (Klaeger/Beklagter), and Gerichtszweig from the user's natural language message
   - Use Akte context when available to improve classification accuracy (e.g., if Akte.sachgebiet is ARBEITSRECHT, prefer Arbeitsrecht-related Klagearten)
   - Map common German legal terms to the correct klageart ID (e.g., "Kuendigungsschutzklage" -> "kschg_klage", "Lohnklage" / "Gehaltsklage" -> "lohnklage", "einstweilige Verfuegung" -> "ev_antrag")
   - Auto-derive gerichtszweig from Rechtsgebiet (ARBEITSRECHT -> ARBG, FAMILIENRECHT -> AG/OLG, VERWALTUNGSRECHT -> VG, etc.)
   - Set confidence < 0.7 when uncertain; include German-language begruendung
   - If the user mentions "aussergerichtlich", "Abmahnung", "Kuendigungsschreiben", "Vergleichsvorschlag", "Aufforderungsschreiben" — set stadium to AUSSERGERICHTLICH

4. Export `recognizeIntent(userMessage: string, akteContext: AkteContext | null): Promise<IntentResult>`:
   - Get tier 3 model via `getModelForTier(3)` (highest quality for intent recognition)
   - Build context block from akteContext if available (Sachgebiet, Kurzrubrum, Beteiligte list)
   - Call generateObject with IntentResultSchema
   - Return parsed result
   - On error, throw with clear message for pipeline error handling

5. Export `buildAkteContext(prisma: PrismaClient, akteId: string): Promise<AkteContext | null>`:
   - Query Akte with beteiligte.kontakt and dokumente
   - Map to AkteContext interface
   - Return null if Akte not found

**slot-filler.ts** — Stage 2 of the pipeline: Slot extraction and Rueckfrage generation.

1. Import `getKlageartDefinition` from "./klageart-registry" and types from "./schemas".

2. Define `SlotValues` type: `Record<string, string | number | boolean | null>` — keys are UPPER_SNAKE_CASE slot keys, values are resolved or null

3. Define `SlotFillResult` interface:
   - slots: SlotValues (all known values including pre-filled and user-provided)
   - missingRequired: SlotDefinition[] (slots that still need values)
   - rueckfrage: string | null (conversational German question for the FIRST missing slot, or null if complete)
   - vollstaendig: boolean (true if all required slots are filled)

4. Export `prefillSlotsFromAkte(prisma: PrismaClient, akteId: string, intent: IntentResult): Promise<Partial<SlotValues>>`:
   - Query Akte with beteiligte (including kontakt with adressen), dokumente, normen
   - Find Mandant and Gegner Beteiligte
   - Map based on intent.rolle:
     - If KLAEGER: mandant->KLAEGER_NAME/KLAEGER_ADRESSE, gegner->BEKLAGTER_NAME/BEKLAGTER_ADRESSE
     - If BEKLAGTER: reverse mapping
   - Pre-fill GERICHT from Beteiligte with rolle "GERICHT" if exists
   - Pre-fill STREITWERT from akte.gegenstandswert if exists
   - Pre-fill SACHGEBIET from akte.sachgebiet
   - Format names using kontakt data (Vorname + Nachname for natural persons, Firma for juristic)
   - Format addresses from kontakt.adressen (first address: strasse, plz, ort)
   - Return partial SlotValues with source attribution comments in a separate field

5. Export `fillSlots(intent: IntentResult, prefilled: Partial<SlotValues>, userProvided: Partial<SlotValues>): SlotFillResult`:
   - Get KlageartDefinition from registry via intent.klageart
   - Merge prefilled + userProvided (user overrides prefilled)
   - Check each requiredSlot against merged values
   - For missing required slots, generate a conversational Rueckfrage for the FIRST one only (one-at-a-time, not batched — per user decision)
   - Rueckfrage format: natural German question, e.g., "Wann wurde die Kuendigung ausgesprochen? (Datum der Kuendigung)" — not a form-like "Bitte geben Sie ein:"
   - Include active warnings for KSchG-specific slots: if ZUGANG_DATUM is provided, calculate and warn about 3-Wochen-Frist deadline
   - If a user answers "weiss ich noch nicht" / "unbekannt" for a slot, set the value to `{{SLOT_KEY}}` placeholder and mark it as filled but unresolved
   - Return SlotFillResult with vollstaendig=true only when all required slots have non-null, non-placeholder values

6. Export `generateRueckfrage(slot: SlotDefinition, intent: IntentResult, existingSlots: SlotValues): string`:
   - Generate a context-aware German question for the given slot
   - Reference the Klageart and existing known information in the question
   - For date slots, hint at expected format (TT.MM.JJJJ)
   - For currency slots, hint at expected format (EUR)
   - Cap at 10 Rueckfragen per pipeline run (caller responsibility, but document the limit)

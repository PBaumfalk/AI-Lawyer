Output: `memory-service.ts` with full lifecycle, wired into both `ki-chat/route.ts` (Chain G) and `helena-task.processor.ts`, with `system-prompt.ts` updated to use structured German markdown formatting.

  <name>Task 1: Create HelenaMemory service with loadOrRefresh, generateMemory, and formatMemoryForPrompt</name>
  <files>src/lib/helena/memory-service.ts</files>
  <action>
Create `src/lib/helena/memory-service.ts` with the following exports:

**1. HelenaMemoryContentSchema (Zod schema):**
Define a Zod schema matching the locked content structure:
- `summary: z.string()` -- detailed case narrative (1-2 paragraphs)
- `risks: z.array(z.string())` -- recognized risks
- `nextSteps: z.array(z.string())` -- pending actions
- `openQuestions: z.array(z.string())` -- unanswered questions
- `relevantNorms: z.array(z.string())` -- legal provisions (e.g., "BGB SS 626")
- `strategy: z.string()` -- lawyer's stated approach
- `keyEvents: z.array(z.object({ date: z.string(), event: z.string() }))` -- case timeline
- `beteiligteNotes: z.record(z.string(), z.string())` -- per-party insights
- `proceduralStatus: z.string()` -- current Verfahrensstand

Export the inferred type as `HelenaMemoryContent`.

**2. formatMemoryForPrompt(content, changes?):**
Pure function that renders `HelenaMemoryContent` into structured German markdown:
- Header: `## Fallgedaechtnis (Stand: DD.MM.YYYY)`
- Section: `### Aenderungen seit letztem Gespraech` (only if `changes[]` provided and non-empty)
- Sections for: Zusammenfassung, Erkannte Risiken, Strategie, Naechste Schritte, Offene Fragen
- Only render sections with content (skip empty arrays/strings)
- Cap total output at ~2000 tokens (~7000 chars) using `estimateTokens()` from `token-budget.ts`. If over budget, truncate the summary field and append "[...]".

**3. generateMemory(prisma, akteId, previousMemory?):**
Internal function that generates memory content via `generateObject` (AI SDK):
- Load Akte data: basic fields + beteiligte (with kontakt) + dokumente (last 30, name/mimeType/createdAt/ordner) + kalenderEintraege (next 20, active) + sachgebiet + status
- Load AiConversation history for this Akte (last 10 conversations, only `messages` and `titel` fields, ordered by updatedAt desc)
- Build a prompt containing Akte structured data + conversation summaries
- Use `getModel()` from `src/lib/ai/provider.ts` for the LLM call
- Use a German system prompt instructing structured case analysis (see RESEARCH.md for recommended prompt)
- Call `generateObject({ model, schema: HelenaMemoryContentSchema, system, prompt })`
- Return the validated `HelenaMemoryContent`

**4. loadOrRefresh(prisma, akteId, options?):**
Main entry point:
- `options?: { cooldownMs?: number }` -- defaults to 5 minutes (300_000ms)
- Step 1: Load `HelenaMemory` + `Akte.geaendert` in a single query:
  ```typescript
  const [memory, akte] = await Promise.all([
    prisma.helenaMemory.findUnique({ where: { akteId } }),
    prisma.akte.findUnique({ where: { id: akteId }, select: { geaendert: true } }),
  ]);
  ```
- Step 2: If no akte found, return `{ content: null, refreshed: false }`
- Step 3: If memory exists and NOT stale (`memory.lastRefreshedAt >= akte.geaendert`) and NOT on first load (memory exists), return existing content with `refreshed: false`
- Step 4: If memory exists but stale, check cooldown. Use an in-memory `Map<string, number>` (module-level, keyed by akteId) to prevent rapid-fire regeneration. If `Date.now() - lastCooldown < cooldownMs`, return existing content with `refreshed: false`
- Step 5: Generate new memory via `generateMemory()`
- Step 6: Detect changes for "Aenderungen" section. Store `_meta` in content: `{ dokumentCount, beteiligteCount, fristenCount }`. Compare current Akte data counts against previous `_meta` to generate change strings (e.g., "3 neue Dokumente", "Neuer Beteiligter hinzugefuegt"). If no `_meta` exists (first refresh), return `["Erste Analyse dieser Akte"]`.
- Step 7: CRITICAL -- Preserve existing `rejectionPatterns` from `HelenaMemory.content`. Load existing content, extract `rejectionPatterns`, merge into new content before upsert.
- Step 8: Upsert with version increment:
  ```typescript
  await prisma.helenaMemory.upsert({
    where: { akteId },
    create: { akteId, content: mergedContent, lastRefreshedAt: new Date() },
    update: { content: mergedContent, version: { increment: 1 }, lastRefreshedAt: new Date() },
  });
  ```
- Step 9: Update cooldown map, clean entries older than 1 hour if map size > 1000
- Step 10: Return `{ content: generatedContent, changes, refreshed: true }`

**Return type:**
```typescript
export interface LoadOrRefreshResult {
  content: HelenaMemoryContent | null;
  changes?: string[];
  refreshed: boolean;
}
```

Use `createLogger("helena-memory")` for logging (import from `@/lib/logger`).
Handle all errors gracefully -- if LLM call fails, return existing memory content (stale but usable) with `refreshed: false` and log the error. Never throw from `loadOrRefresh()`.

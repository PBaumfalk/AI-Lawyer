```typescript
// Filter params accepted by all BI endpoints
export interface BiFilterParams {
  zeitraum: "monat" | "quartal" | "jahr" | "custom";
  von?: string;     // ISO date
  bis?: string;     // ISO date
  anwaltId?: string;
  sachgebiet?: string; // Sachgebiet enum value
}

// Single KPI tile data
export interface KpiTile {
  id: string;           // e.g. "akten-offen"
  label: string;        // German display label
  value: number;
  previousValue: number;
  delta: number;         // percentage change (positive = up)
  unit?: string;         // "EUR", "%", or omit for count
  domain: "akten" | "finanzen" | "fristen" | "helena";
}

// Full KPI response
export interface BiKpiResponse {
  tiles: KpiTile[];
  zeitraum: { von: string; bis: string };
  cached: boolean;
}

// Trend data point
export interface TrendPoint {
  label: string;    // "Jan 2026", "Feb 2026" etc.
  date: string;     // ISO date of period start
  value: number;
}

// Single trend series
export interface TrendSeries {
  id: string;           // e.g. "akten-neuzugang"
  label: string;
  data: TrendPoint[];
  type: "line" | "area";
}

// Trend response
export interface BiTrendResponse {
  series: TrendSeries[];
  zeitraum: { von: string; bis: string };
  cached: boolean;
}
```

Create `src/lib/bi/cache.ts` with Redis cache wrapper:
- Import `createRedisConnection` from `@/lib/redis`
- Export `cachedQuery<T>(key: string, ttlSeconds: number, queryFn: () => Promise<T>): Promise<{ data: T; cached: boolean }>`
- On cache hit: parse JSON, return `{ data, cached: true }`
- On cache miss: execute queryFn, store result with SETEX (default 300s = 5min), return `{ data, cached: false }`
- Key prefix: `bi:` to namespace BI cache keys
- Handle Redis errors gracefully (log warning, fall through to queryFn)

Create `src/lib/bi/kpi-queries.ts`:

**Helper:** `parseBiFilters(searchParams: URLSearchParams): BiFilterParams` -- parse zeitraum, von, bis, anwaltId, sachgebiet from query string. Default zeitraum="monat". Compute date range from zeitraum (monat = current month, quartal = current quarter start, jahr = Jan 1 of current year, custom = von/bis params).

Also compute "previous period" range for delta calculation (e.g., if current = this month, previous = last month).

**`getAktenKpis(filters, accessFilter)`:** Returns KpiTile[] with:
- `akten-offen`: Count Akte where status=OFFEN in period (filtered by sachgebiet/anwaltId if set)
- `akten-neuzugang`: Count Akte where erstellt >= period start
- Apply accessFilter from RBAC

**`getFinanzenKpis(filters, accessFilter)`:** Returns KpiTile[] with:
- `finanzen-umsatz`: Sum Rechnung.gesamtBrutto where rechnungsDatum in period
- `finanzen-offene-rechnungen`: Count Rechnung where status=OFFEN (or bezahltAm is null)
- Apply accessFilter via akte relation

**`getFristenKpis(filters, accessFilter)`:** Returns KpiTile[] with:
- `fristen-compliance`: Percentage of KalenderEintrag (typ=FRIST) that are erledigt AND erledigtAm <= datum (completed before deadline) vs total in period
- `fristen-ueberfaellig`: Count overdue (not erledigt, datum < now)
- Apply accessFilter via akte relation

**`getHelenaKpis(filters, accessFilter)`:** Returns KpiTile[] with:
- `helena-gespraeche`: Count HelenaTask in period
- `helena-entwuerfe`: Count HelenaTask where type involves draft (or has output)
- `helena-akzeptanzrate`: Percentage of HelenaTask where status=ACCEPTED vs total completed
- `helena-token`: Sum tokenUsage from HelenaTask in period (check schema for actual field name -- may need to aggregate from JSON metadata)

Each function: query current period AND previous period, compute delta as `((current - previous) / previous) * 100`. Handle division by zero (delta = 0 if previous = 0 and current = 0, delta = 100 if previous = 0 and current > 0).

**`getTrendData(filters, accessFilter)`:** Returns TrendSeries[] with monthly data points for last 12 months (or period from filters):
- `akten-neuzugang`: Monthly count of new Akte
- `umsatz-monat`: Monthly sum of Rechnung.gesamtBrutto
- `fristen-compliance`: Monthly compliance rate

Use Prisma `groupBy` with date truncation or raw SQL `DATE_TRUNC('month', ...)` for monthly aggregation.

Wrap ALL query functions in `cachedQuery` from cache.ts. Cache key format: `bi:{function}:{filterHash}:{userId}` where filterHash is a stable JSON hash of filter params.

Create `src/app/api/bi/kpis/route.ts`:
- GET handler with `requireAuth` + `requirePermission("finanzen", "read")` or just requireAuth
- Parse query params into BiFilterParams
- Build accessFilter via `buildAkteAccessFilter(session.user.id, session.user.role)`
- Call all four KPI functions in Promise.all
- Return `BiKpiResponse` with merged tiles array

Create `src/app/api/bi/trends/route.ts`:
- GET handler with requireAuth
- Parse query params into BiFilterParams
- Call getTrendData
- Return `BiTrendResponse`

Both endpoints: return 401 if not authenticated, catch errors and return 500 with message.

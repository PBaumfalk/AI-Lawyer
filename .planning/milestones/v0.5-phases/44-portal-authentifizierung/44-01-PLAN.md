**1. Prisma Schema — Add PortalInvite model** (after Beteiligter model, ~line 878):

```prisma
model PortalInvite {
  id            String    @id @default(cuid())
  token         String    @unique @default(cuid())
  beteiligteId  String
  beteiligter   Beteiligter @relation(fields: [beteiligteId], references: [id], onDelete: Cascade)
  kontaktId     String
  kontakt       Kontakt   @relation(fields: [kontaktId], references: [id])
  email         String    // Email to which invite was sent
  status        PortalInviteStatus @default(PENDING)
  expiresAt     DateTime
  createdById   String
  createdBy     User      @relation("PortalInviteCreator", fields: [createdById], references: [id])
  createdAt     DateTime  @default(now())
  usedAt        DateTime?

  @@map("portal_invites")
}

enum PortalInviteStatus {
  PENDING
  ACCEPTED
  EXPIRED
  REVOKED
}
```

Add relations:
- On Beteiligter: `portalInvites PortalInvite[]`
- On Kontakt: `portalInvites PortalInvite[]`
- On User: `portalInvitesCreated PortalInvite[] @relation("PortalInviteCreator")`

Run `npx prisma db push` to apply schema changes.

**2. Email template** — `src/lib/email/templates/portal-invite.ts`:

Export a function `portalInviteEmail(params: { mandantName: string; kanzleiName: string; aktenzeichen: string; activationUrl: string }): { subject: string; text: string; html: string }`.

- Subject: "Einladung zum Mandantenportal — {kanzleiName}"

**3. POST /api/portal/invite** — `src/app/api/portal/invite/route.ts`:

- Auth: require session with role ANWALT or ADMIN (reject SACHBEARBEITER/SEKRETARIAT/MANDANT)
- Request body: `{ beteiligteId: string }`
- Validation:
  - Beteiligter must exist and have rolle=MANDANT
  - Kontakt must have an email address (return 400 if not)
  - If Kontakt already has a portal User (check for existing User with role=MANDANT and matching kontaktId), return 400 "Mandant hat bereits einen Portal-Account"
  - If an active (PENDING, not expired) invite already exists for this Beteiligter, revoke it before creating new one
- Create PortalInvite with:
  - token: Use `crypto.randomUUID()` for the token (more secure than cuid)
  - expiresAt: 7 days from now
  - email: kontakt.email
  - kontaktId: from Beteiligter
  - createdById: session.user.id
- Build activation URL: `${process.env.NEXTAUTH_URL}/portal/activate?token={token}`
- Send invite email using sendEmail() + portalInviteEmail template
- Look up Kanzlei name from the Akte's kanzleiId and Aktenzeichen from the Akte
- Log audit event: `logAuditEvent({ userId: session.user.id, aktion: "PORTAL_EINLADUNG", details: { kontaktId, beteiligteId, email } })`
- Return 201 with `{ id, token, email, expiresAt }`

**4. POST /api/portal/invite/[id]/resend** — `src/app/api/portal/invite/[id]/resend/route.ts`:

- Auth: require ANWALT or ADMIN
- Find the PortalInvite by id, check it's PENDING and not expired
- If expired, update status to EXPIRED and create a new invite (same beteiligteId/kontaktId), send new email
- If still valid, just resend the email
- Return 200 with updated invite

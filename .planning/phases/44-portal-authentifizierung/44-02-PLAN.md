**1. POST /api/portal/activate** — `src/app/api/portal/activate/route.ts`:

This is a PUBLIC endpoint (no auth required — Mandant doesn't have an account yet).

Request body: `{ token: string; password: string; passwordConfirm: string }`

Validation:
- token must be a non-empty string
- password must be min 8 chars, at least 1 uppercase letter, at least 1 number (regex: `/^(?=.*[A-Z])(?=.*\d).{8,}$/`)
- password must equal passwordConfirm

Logic:
1. Find PortalInvite by token where status=PENDING
2. If not found or expired (expiresAt < now), return 400 "Einladung ungültig oder abgelaufen"
3. Check if a User with role=MANDANT and kontaktId=invite.kontaktId already exists — if so, return 400 "Account existiert bereits"
4. Create new User:
   - email: invite.email
   - name: Build from Kontakt (vorname + nachname, or firma) — query invite.kontakt for this
   - passwordHash: bcrypt.hash(password, 12)
   - role: "MANDANT" (UserRole enum)
   - aktiv: true
   - kontaktId: invite.kontaktId (Phase 43 should have added this FK)
5. Update PortalInvite: status=ACCEPTED, usedAt=now()
6. Log audit event: `logAuditEvent({ aktion: "PORTAL_AKTIVIERT", details: { kontaktId, email, userId: newUser.id } })`
7. Return 201 with `{ success: true, email: invite.email }` (don't auto-login — redirect to login page)

Create a client-side `PortalSessionProvider` (React context + provider):
- Wraps portal layout children
- Tracks user activity via event listeners: mousemove, keydown, click, scroll, touchstart
- Uses a timer (setTimeout) that resets on activity
- After 30 minutes (1800000ms) of no activity, calls `signOut({ callbackUrl: "/portal/login" })`
- Show a warning dialog at 25 minutes: "Ihre Sitzung läuft in 5 Minuten ab. Klicken Sie um fortzufahren."
- If user clicks "Weiter", reset the timer
- Debounce activity events (throttle to once per 30s to avoid excessive resets)
- Export `PortalSessionProvider` component

### Verified Integration Points

1. **Phase 33 -> 34** (Profile -> Dashboard): `/api/gamification/dashboard` calls `getOrCreateGameProfile()` + `evaluateQuestCondition()` from Phase 33 services. Opt-in gate consistent (DB query, not session token).
2. **Phase 33 -> 35** (Quest -> Boss): Shared `gamificationQueue` handles quest-check and boss-damage/heal/check jobs. Shared `awardRewards()` for atomic XP/Runen increments. Nightly safety net also triggers `checkAndSpawnBoss`.
3. **Phase 34 + 35 -> Dashboard**: `QuestWidget` and `BossfightBanner` both render on dashboard without conflict. Both use absent-until-loaded pattern.
4. **Phase 36** (Independent): `AkteDetailClient` and `OcrRecoveryBanner` self-contained within akte detail. Both recovery APIs wire into embedding pipeline correctly.
5. **Socket.IO**: `SocketProvider` wraps dashboard layout. `setupRooms` auto-joins `kanzlei:{id}` room. Boss engine emits to correct room. All 5 event shapes match consumer contracts (verified after Plan 35-03 gap closure).

### Integration Issues (2)

**1. Missing `id` field in bossfight API recentDamage response** (Medium)
- **From:** `GET /api/gamification/bossfight` -> **To:** `BossDamageTicker`
- **File:** `src/app/api/gamification/bossfight/route.ts` lines 89-95
- **Issue:** API maps `recentDamage` without `id` field; `BossDamageTicker` uses `entry.id` as React key in `AnimatePresence`; on initial page load, all entries get `key={undefined}` causing React key collision
- **Fix:** Add `id: d.id` to mapped recentDamage objects
- **Affected:** BOSS-03

**2. Generic PATCH kalender route skips boss damage** (Low)
- **From:** `PATCH /api/kalender/[id]` -> **To:** `boss-engine`
- **File:** `src/app/api/kalender/[id]/route.ts` lines 134-137
- **Issue:** Fires `enqueueQuestCheck` but not `boss-damage` job for WV completions; the dedicated `/erledigt` sub-route handles both correctly
- **Risk:** Only triggers if UI uses generic PATCH instead of dedicated endpoint for WV completion
- **Affected:** BOSS-01

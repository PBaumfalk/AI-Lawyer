**1. Create GET /api/gamification/dashboard endpoint** at `src/app/api/gamification/dashboard/route.ts`:

- Auth check via `auth()` -- return 401 if no session
- Query `gamificationOptIn` from `prisma.user.findUnique({ where: { id }, select: { gamificationOptIn: true, role: true } })` (same pattern as existing profile route -- do NOT read from session token)
- If `!gamificationOptIn`, return 404 (widget uses this to know user is opted out)
- Call `getOrCreateGameProfile(userId, role)` for XP/level/runen/streak data
- Compute level, levelTitle, progress using `getLevelForXp`, `getLevelTitle`, `getRequiredXp` (same math as existing profile route)
- Load active daily quests: `prisma.quest.findMany({ where: { typ: "DAILY", aktiv: true }, orderBy: { sortierung: "asc" } })`
- For each quest, call `evaluateQuestCondition(condition, userId)` for real-time current/target/completed counts
- For each quest, check if `QuestCompletion` exists for today (awarded flag): `prisma.questCompletion.findFirst({ where: { userId, questId, completedAt: { gte: startOfDay(new Date()) } } })`
- Use `Promise.all` for quest evaluation + completion checks (parallel per quest)
- Return JSON response shape:
```typescript
{
  profile: {
    level: number;
    levelTitle: string;
    xp: number;
    xpInLevel: number;
    xpNeeded: number;
    progress: number; // 0-1, rounded to 2 decimals
    runen: number;
    streakTage: number;
  };
  quests: Array<{
    id: string;
    name: string;
    beschreibung: string;
    bedingung: QuestCondition;
    xpBelohnung: number;
    runenBelohnung: number;
    current: number;
    target: number;
    completed: boolean;
    awarded: boolean;
  }>;
}
```

**2. Create buildQuestDeepLink utility** at `src/components/gamification/quest-deep-link.ts`:

- Import `QuestCondition` from `@/lib/gamification/types`
- Static MODEL_TO_PATH lookup: `KalenderEintrag -> "/kalender"`, `Ticket -> "/tickets"`, `Rechnung -> "/finanzen/rechnungen"`, `AktenActivity -> "/akten"`
- Build URL from condition: base path from model, query params from `condition.where` fields (typ, status, erledigt), `condition.period === "today"` adds `datum=heute`
- Fallback to "/dashboard" for unknown models
- Export `buildQuestDeepLink(condition: QuestCondition): string`

**2. Create QuestWidget** at `src/components/gamification/quest-widget.tsx`:

- `"use client"` component, no props needed (self-fetching pattern like Tagesuebersicht)
- State: `data` (response type or null), `error` boolean
- `useEffect` fetches `/api/gamification/dashboard`:
  - If response status is 404 -> set data to null (user opted out, render nothing)
  - If response not ok -> set error true, `console.error("QuestWidget: failed to load gamification data")`
  - Otherwise parse JSON and set data
- **Graceful degradation**: if `error || data === null`, return `null` (widget does not render, no placeholder, no empty space)
- Use "absent-until-loaded" pattern: return `null` until first fetch completes (avoids flicker)
- Wrap in `<GlassCard className="p-0">` (same pattern as dashboard cards)

- **Header section** (inside a `px-5 py-4 border-b border-[var(--glass-border-color)]` div):
  - Left side: level title (`text-sm font-semibold`) + "Lv. {level}" (`text-xs text-foreground/70`)
  - Right side: inline pill badges for streak and runen:

  - Below title row: `<XpProgressBar progress={data.profile.progress} xpCurrent={data.profile.xpInLevel} xpNeeded={data.profile.xpNeeded} />`

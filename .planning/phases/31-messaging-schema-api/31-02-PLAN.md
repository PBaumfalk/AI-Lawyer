  <action>
**1. Create src/app/api/channels/route.ts** with GET and POST handlers:

**GET /api/channels** -- List channels the user is a member of:
- requireAuth()
- Query ChannelMember where userId = session.user.id, include channel with _count of members and messages
- For each channel, compute unreadCount via getUnreadCount()
- Also include the most recent message createdAt as lastMessageAt (or null)
- Sort by lastMessageAt desc (most recently active first), then by name
- Filter: optional `?typ=ALLGEMEIN|AKTE` query param
- Also support `?browse=true` to list ALL non-archived ALLGEMEIN channels (for browsing channels to join), including a `joined: boolean` flag
- Return: `{ channels: ChannelListItem[] }`

**POST /api/channels** -- Create a new ALLGEMEIN channel:
- requireAuth()
- Zod schema: `{ name: z.string().min(1).max(100).trim(), beschreibung: z.string().max(500).optional() }`
- Call createChannel() from channel-service with erstelltVonId = session.user.id
- If slug collision (P2002 error), return 409 with "Ein Kanal mit diesem Namen existiert bereits"
- Return: `{ channel }` with status 201

**2. Create src/app/api/channels/[id]/route.ts** with GET, PATCH, DELETE handlers:

**GET /api/channels/{id}** -- Channel detail:
- requireAuth()
- Find channel by id, include members with user info (id, name, avatarUrl), include _count of messages
- Verify user is a member (ChannelMember exists) OR user is ADMIN. For AKTE channels, also validate via requireAkteAccess() using channel.akteId.
- If not a member and not ADMIN, return 404.
- Return: `{ channel }` including members array and message count

**PATCH /api/channels/{id}** -- Update channel name/description:
- requireAuth()
- Zod schema: `{ name: z.string().min(1).max(100).trim().optional(), beschreibung: z.string().max(500).optional() }`
- Only the channel creator (erstelltVonId) or ADMIN can update
- If name changes, regenerate slug. Handle slug collision with 409.
- Cannot update AKTE channels (they derive name from Akte kurzrubrum)
- Return: `{ channel }`

**DELETE /api/channels/{id}** -- Archive (soft-delete) channel:
- requireAuth()
- Only creator or ADMIN can archive
- Set archived = true (not actual delete)
- Cannot archive AKTE channels (they live as long as the Akte)
- Return: `{ success: true }`

**1. Create src/app/api/channels/[id]/members/route.ts** with GET, POST, DELETE:

**GET /api/channels/{id}/members** -- List channel members:
- requireAuth()
- Verify user is member or ADMIN
- Return: `{ members: [{ id, userId, userName, userAvatarUrl, joinedAt, lastReadAt }] }`

**POST /api/channels/{id}/members** -- Join channel:
- requireAuth()
- Only works for ALLGEMEIN channels (AKTE membership is managed by RBAC sync)
- Check channel exists and is not archived
- Check channel.typ === "ALLGEMEIN" (return 400 for AKTE: "Akte-Kanaele werden automatisch verwaltet")
- Create ChannelMember with userId = session.user.id, lastReadAt = now
- Handle duplicate (P2002) gracefully -- return 200 "Bereits Mitglied"
- Return: `{ member }` with status 201

**DELETE /api/channels/{id}/members** -- Leave channel:
- requireAuth()
- Only works for ALLGEMEIN channels
- Delete ChannelMember where channelId + userId
- If not found, return 404
- Return: `{ success: true }`

**2. Create src/app/api/channels/[id]/read/route.ts** with PATCH:

**PATCH /api/channels/{id}/read** -- Mark channel as read:
- requireAuth()
- updateMany ChannelMember where channelId + userId, set lastReadAt = new Date()
- If updated.count === 0, return 403 "Nicht Mitglied dieses Kanals"
- Return: `{ success: true }`

**3. Create src/app/api/akten/[id]/channel/route.ts** with GET:

**GET /api/akten/{id}/channel** -- Lazy-create AKTE channel:
- requireAkteAccess(akteId) -- gates access via standard RBAC
- Look up existing channel: prisma.channel.findUnique({ where: { akteId } })
- If channel exists:
  - Call syncAkteChannelMembers(channel.id, akteId) to ensure RBAC members are up-to-date
  - Return channel with members
- If no channel:
  - Create in $transaction:
    1. Create Channel with name = `Akte: ${akte.kurzrubrum || akteId}`, slug = `akte-${akteId}`, typ = "AKTE", akteId, erstelltVonId = session.user.id
    2. Call gatherAkteMemberIds(tx, akteId) to get member IDs
    3. createMany ChannelMember for all gathered IDs with skipDuplicates
  - Return newly created channel with members
- Handle race condition: catch P2002 on Channel.akteId unique constraint, retry as findUnique
- Include members with user info (id, name, avatarUrl) in response
- Return: `{ channel }` (with members array)

Note: The `akte` object returned from requireAkteAccess() only has a limited select (id, anwaltId, sachbearbeiterId, kanzleiId). For kurzrubrum in the channel name, do a separate findUnique to fetch kurzrubrum.

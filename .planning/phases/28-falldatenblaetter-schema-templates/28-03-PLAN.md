<!-- API contract from Plan 28-02 -->

GET /api/falldaten-templates -> Template[] (with erstelltVon, status, sachgebiet, schema)
POST /api/falldaten-templates -> Template (body: { name, beschreibung?, sachgebiet?, schema: { felder: [...] } })

PATCH /api/falldaten-templates/[id] -> Updated template
DELETE /api/falldaten-templates/[id] -> 204
POST /api/falldaten-templates/[id]/einreichen -> Updated template (EINGEREICHT)
POST /api/falldaten-templates/[id]/genehmigen -> Updated template (GENEHMIGT)
POST /api/falldaten-templates/[id]/ablehnen -> Updated template (body: { ablehnungsgrund })

```typescript
interface TemplateResponse {
  id: string;
  name: string;
  beschreibung: string | null;
  sachgebiet: string | null;

    key: string; label: string; typ: string;
    placeholder?: string; optionen?: { value: string; label: string }[];
    required?: boolean; gruppe?: string;
  }> };
  version: number;
  status: "ENTWURF" | "EINGEREICHT" | "GENEHMIGT" | "ABGELEHNT" | "STANDARD";
  erstelltVonId: string;
  erstelltVon: { id: string; name: string };
  geprueftVonId: string | null;
  geprueftVon: { name: string } | null;
  geprueftAt: string | null;
  ablehnungsgrund: string | null;
  createdAt: string;

}
```

**Section 2 — Gruppen & Felder Builder:**

State model:
```typescript
interface BuilderGroup {
  name: string;
  fields: BuilderField[];
}
interface BuilderField {
  key: string;
  label: string;
  typ: FalldatenFeldTypDB;
  placeholder: string;
  required: boolean;
  optionen: { value: string; label: string }[];
}
// State: groups: BuilderGroup[]
```

Group management:
- "Gruppe hinzufuegen" button -> shows input for group name, adds to list
- Each group rendered as a collapsible card with group name as header
- Group header has a delete button (only if no fields in group, or confirm if fields exist)

Field management within each group:
- "Feld hinzufuegen" button inside each group card
- Clicking opens an inline form (not modal) with:
  - Label (required text input)
  - Key (auto-generated from label: lowercase, replace spaces/special chars with underscore, trim to 64 chars — editable by user)
  - Typ (dropdown: Text, Textbereich, Zahl, Waehrung, Datum, Dropdown, Checkbox, Mehrfachauswahl — map to internal typ values)
  - Pflichtfeld toggle (checkbox, default false)
  - Placeholder (optional text input, shown only for text/textarea/number/currency types)
  - Optionen section (shown only when typ is "select" or "multiselect"):
    - List of value/label pairs with add/remove buttons
    - "Option hinzufuegen" button adds empty row
    - Each row: value input + label input + remove button
- "Feld speichern" button adds field to the group's fields array
- Each existing field shown as a compact row: Label (Typ) [Pflicht] with edit/delete icons
- Field ordering: fixed by creation order within group (per CONTEXT decision)

**Typ label mapping for the dropdown (German labels):**
- text -> "Text"
- textarea -> "Textbereich"
- number -> "Zahl"
- currency -> "Waehrung"
- date -> "Datum"
- select -> "Dropdown"
- boolean -> "Checkbox"
- multiselect -> "Mehrfachauswahl"

**Section 3 — Actions:**
- "Als Entwurf speichern" button (POST to /api/falldaten-templates, then redirect to list page)
- "Speichern und einreichen" button (POST create, then POST einreichen, redirect to list)
- "Abbrechen" link (back to list)

On save, transform state into API payload:
```typescript
const schema = {
  felder: groups.flatMap(group =>
    group.fields.map(field => ({
      key: field.key,
      label: field.label,
      typ: field.typ,
      placeholder: field.placeholder || undefined,
      required: field.required || undefined,
      gruppe: group.name,
      optionen: field.optionen.length > 0 ? field.optionen : undefined,
    }))
  ),
};
```

Show validation errors from the API response (Zod errors) as inline messages.

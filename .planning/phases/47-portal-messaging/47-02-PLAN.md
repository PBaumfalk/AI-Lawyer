  - Opens file picker (hidden `<input type="file" multiple>`)

  - Max 5 files per message
  - Selected files shown as removable chips below the textarea (file name + X button to remove)
- Send button (SendHorizontal icon or ArrowUp):
  - Disabled when body is empty AND no attachments
  - On click or Enter:
    1. If attachments exist, upload each file first:
       - POST each file to a portal upload endpoint (see below)
       - Collect the returned dokumentId and name for each file
    2. POST to `/api/portal/akten/${akteId}/messages` with body + attachments array
    3. Call onMessageSent() to trigger message list refetch
    4. Clear input and attachments
  - Show loading spinner on send button while sending
- NO @mention picker (Mandant doesn't need mentions)
- NO @Helena button (Portal messages don't support Helena)
- Glass UI styling: glass-card-like input area, subtle border-top separator

**2. Update/create file upload endpoint for portal:**
- Check if there's already a portal document upload endpoint from Phase 46 (DOC-04 Mandant upload). If so, reuse it for message attachments.
- If not, create `POST /api/portal/akten/[id]/upload` or extend the messages route:
  - Auth: requirePortalAuth()
  - Accept multipart form data (use `request.formData()`)
  - For each file:
    - Validate file size (max 25 MB) and type
    - Generate MinIO key: `portal/${akteId}/nachrichten/${cuid()}-${filename}`
    - Upload to MinIO via `uploadFile()` from src/lib/storage.ts
    - Create a Dokument record in Prisma (linked to the Akte, kategorie: "KORRESPONDENZ" or "SONSTIGES", creator: MandantUser's linked userId)
    - Return `{ dokumentId, name, url }` (url = signed download URL)
  - Alternative simpler approach: Upload files as part of the message send flow. The POST /api/portal/akten/[id]/messages endpoint already accepts `attachments` array. Extend it to also accept file uploads (multipart). Or keep it as a two-step flow: upload files first to get dokumentIds, then send message with attachment references.
  - RECOMMENDED: Use the existing DMS upload pattern from Phase 46 if available. If Phase 46's Mandant upload endpoint exists at `/api/portal/akten/[id]/dokumente`, upload files there first, get dokumentIds, then include as attachments in the message POST.

**Time range:** Last calendar month. E.g., if today is March 2026, export covers February 2026.

**Data gathering (all server-side):**

1. **Backlog-Delta per week:** Get WeeklySnapshot entries for model="Wiedervorlage" within last month. For each week, calculate delta from previous week. If no previous week exists, show just the count.

2. **Billing-Delta:** Compare Rechnung aggregates:
   ```typescript
   const lastMonth = subMonths(new Date(), 1);
   const monthStart = startOfMonth(lastMonth);
   const monthEnd = endOfMonth(lastMonth);
   const prevMonthStart = startOfMonth(subMonths(lastMonth, 1));
   const prevMonthEnd = endOfMonth(subMonths(lastMonth, 1));

   const [thisMonthBilling, prevMonthBilling] = await Promise.all([
     prisma.rechnung.aggregate({
       where: {
         akte: { kanzleiId },
         rechnungsdatum: { gte: monthStart, lte: monthEnd },
         status: { not: "STORNIERT" },
       },
       _sum: { betragNetto: true },
       _count: { id: true },
     }),
     prisma.rechnung.aggregate({
       where: {
         akte: { kanzleiId },
         rechnungsdatum: { gte: prevMonthStart, lte: prevMonthEnd },
         status: { not: "STORNIERT" },
       },
       _sum: { betragNetto: true },
       _count: { id: true },
     }),
   ]);
   ```
   Calculate delta percent: `lastTotal > 0 ? Math.round(((thisTotal - lastTotal) / lastTotal) * 100) : (thisTotal > 0 ? 100 : 0)`

3. **Quest fulfillment rate for the month:** Count QuestCompletion entries for DAILY quests within the month, divided by (opted-in users count * active daily quests count * workdays in month). Simplified: use the same per-user average approach but across the month's completions.
   - Get all opted-in users with classes
   - Get all active daily quests
   - Count completions per user for the month
   - For each user: completions / (available quests * workdays in month), capped at 1.0
   - Average across users, round to percentage

   **Simplification (RECOMMENDED):** Count distinct completedDate entries per user in the month. For each user: `distinctDays / totalWorkdays * 100` gives daily engagement rate. OR more simply: count total completions / (available quests * opted-in users * workdays). This captures the monthly aggregate without over-complicating.

   **Final approach:** Same formula as live dashboard but averaged over the month's working days. Count total quest completions in month, divide by (available quests * opted-in users * working days), multiply by 100.

4. **Kanzlei data for Briefkopf:** `prisma.kanzlei.findUnique({ where: { id: kanzleiId } })`

**CSV generation:**

One row per week in the month from WeeklySnapshot data.

Filename: `team-report-{MonthName}-{Year}.csv` (e.g., `team-report-Februar-2026.csv`)

7. **Footer:** Generation timestamp

Use consistent layout: A4 portrait, ~40-57pt margins, 9pt body font, 8pt table font.

Handle edge cases:

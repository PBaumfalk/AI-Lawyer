    **Update src/middleware.ts to enforce 2FA:**

    The current middleware uses `auth` from NextAuth which calls the `authorized` callback in auth.config.ts. The authorized callback is the right place to add 2FA enforcement.

    However, auth.config.ts is Edge-runtime compatible (no Prisma). The `auth.totp.requiredRoles` setting is in the DB — can't be fetched in Edge middleware.

    **Solution:** Store the required roles setting in an environment variable OR use a cached approach.

    **ACTUAL SOLUTION:** Use the NextAuth authorized callback in auth.config.ts to read `token.totpEnabled` and check against a hardcoded ENV var `TOTP_REQUIRED_ROLES`. The admin can set this via system settings which writes to `.env.local` OR we fetch it from a public API route.

    **SIMPLEST working solution:** The authorized callback in auth.config.ts gets the token. We check:
    1. Is the token's role in the required roles? (from `process.env.TOTP_REQUIRED_ROLES || ""`)
    2. Is token.totpEnabled false?
    3. If both: redirect to /2fa-setup-required

    Admins configure TOTP_REQUIRED_ROLES via the admin settings UI (which writes to .env or uses a separate mechanism).

    Since the existing SystemSetting approach writes to DB (not env), and middleware is Edge, use a different approach:

    **FINAL APPROACH:** Extend auth.config.ts authorized callback to read from the token itself:
    ```typescript
    authorized({ auth, request }) {
      // auth here is the session/token from JWT
      const token = auth as any;  // contains role, totpEnabled
      const url = request.nextUrl;

      // Allow 2fa-setup-required page itself
      if (url.pathname === "/2fa-setup-required") return true;

      // 2FA enforcement check (roles configured via env var)
      const requiredRolesEnv = process.env.TOTP_REQUIRED_ROLES || "";
      if (requiredRolesEnv && token?.role && !token?.totpEnabled) {
        const requiredRoles = requiredRolesEnv.split(",").map((r: string) => r.trim());
        if (requiredRoles.includes(token.role)) {
          return Response.redirect(new URL("/2fa-setup-required", request.url));
        }
      }

      // Existing auth check
      return !!auth;
    }
    ```

    Add to authConfig (auth.config.ts) — the authorized callback is added here since auth.config.ts is Edge-compatible.

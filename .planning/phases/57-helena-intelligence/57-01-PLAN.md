Implement Falldatenblatt auto-fill: Helena extracts structured data from Akte documents into Falldatenblatt fields using AI, showing per-field confidence and source excerpts. Users review and accept/reject each suggestion individually -- nothing is auto-saved.

Purpose: Lawyers spend significant time manually transferring data from documents into case data sheets. This automates extraction while keeping the lawyer in control (BRAK 2025 compliance: AI output is always VORSCHLAG).

Output: API endpoint for extraction, AI extraction service, and review UI component integrated into the existing Falldaten tab.

Create `src/lib/helena/falldaten-extractor.ts`:

1. Export interface `FalldatenSuggestion`:
   ```typescript
   export interface FalldatenSuggestion {
     key: string;          // field key from template
     value: any;           // extracted value (string, number, boolean, date string)
     konfidenz: "HOCH" | "MITTEL" | "NIEDRIG";
     quellExcerpt: string; // max 200 chars from source document
     dokumentName: string;  // name of source document
   }
   ```

2. Export async function `extractFalldaten(options)`:
   - Accept: `{ prisma, akteId, felder: TemplateField[], userId: string }`
   - Load the Akte's documents that have ocrText (ocrStatus === "ABGESCHLOSSEN"), take max 10 most recent
   - For each document, fetch LEFT("ocrText", 3000) via raw query (same pattern as ki-chat route)
   - Concatenate document texts with document name headers into a single context string (max 15000 chars total)
   - Build a Zod schema dynamically from the template `felder` array:
     - For each field, create a z.object with: `value` (z.string/z.number/z.boolean depending on feld.typ), `konfidenz` (z.enum(["HOCH", "MITTEL", "NIEDRIG"])), `quellExcerpt` (z.string)
     - The overall schema is z.object with field keys mapping to the per-field schema, all .optional()
   - Call `generateObject` from "ai" SDK with:
     - model from getModel()
     - system prompt in German: "Du bist Helena, KI-Assistentin einer Kanzlei. Extrahiere strukturierte Falldaten aus den folgenden Dokumenten. Fuer jedes Feld: gib den extrahierten Wert, eine Konfidenz-Einschaetzung (HOCH wenn klar im Text, MITTEL wenn abgeleitet, NIEDRIG wenn unsicher/geraten), und ein kurzes Zitat aus dem Quelldokument (max 200 Zeichen). Wenn ein Feld nicht aus den Dokumenten ableitbar ist, lasse es weg."
     - Include the document texts and the field definitions (label, typ, optionen if select) in the prompt
     - schema: the dynamically built Zod schema
   - Map the generateObject result to `FalldatenSuggestion[]`, adding `dokumentName` from the document headers
   - Track token usage via `trackTokenUsage` with funktion "FALLDATEN_AUTOFILL" (add this to the AiFunktion type if needed -- check src/lib/ai/token-tracker.ts; if it's an enum, just use "CHAT" instead)
   - Return the suggestions array

3. Create `src/app/api/akten/[id]/falldaten-autofill/route.ts`:
   - POST handler with requireAuth + requireAkteAccess (follow pattern from other akten/[id] endpoints)
   - Load the Akte with its falldatenTemplate (include template schema)
   - If no falldatenTemplate assigned, return 400 "Kein Falldatenblatt-Template zugewiesen"
   - Extract template felder from akte.falldatenTemplate.schema.felder
   - Call extractFalldaten with the felder
   - Return JSON: `{ suggestions: FalldatenSuggestion[] }`
   - Handle errors with try/catch, return 500 on failure

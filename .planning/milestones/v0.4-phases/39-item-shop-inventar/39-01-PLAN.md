  **2. Manual Migration SQL (prisma/migrations/manual_item_shop.sql)**

  Create idempotent migration:
  ```sql
  -- Phase 39: Item-Shop + Inventar Schema
  BEGIN;

  -- 1. Add FOKUSZEIT to KalenderTyp enum
  DO $$ BEGIN
    ALTER TYPE "KalenderTyp" ADD VALUE IF NOT EXISTS 'FOKUSZEIT';
  EXCEPTION WHEN duplicate_object THEN NULL;
  END $$;

  -- 2. Create ItemRarity enum
  DO $$ BEGIN
    CREATE TYPE "ItemRarity" AS ENUM ('COMMON', 'RARE', 'EPIC', 'LEGENDARY');
  EXCEPTION WHEN duplicate_object THEN NULL;
  END $$;

  -- 3. Create ItemTyp enum
  DO $$ BEGIN
    CREATE TYPE "ItemTyp" AS ENUM ('AVATAR_RAHMEN', 'BANNER', 'PROFIL_TITEL', 'ABSCHLUSS_ANIMATION', 'PERK');
  EXCEPTION WHEN duplicate_object THEN NULL;
  END $$;

  -- 4. Create shop_items table
  CREATE TABLE IF NOT EXISTS "shop_items" (
    "id" TEXT NOT NULL,
    "slug" TEXT NOT NULL,
    "name" TEXT NOT NULL,
    "beschreibung" TEXT,
    "typ" "ItemTyp" NOT NULL,
    "rarity" "ItemRarity" NOT NULL,
    "preis" INTEGER NOT NULL,
    "aktiv" BOOLEAN NOT NULL DEFAULT true,
    "sortierung" INTEGER NOT NULL DEFAULT 0,
    "metadata" JSONB NOT NULL DEFAULT '{}',
    "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "updatedAt" TIMESTAMP(3) NOT NULL,

    CONSTRAINT "shop_items_pkey" PRIMARY KEY ("id")
  );

  CREATE UNIQUE INDEX IF NOT EXISTS "shop_items_slug_key" ON "shop_items"("slug");
  CREATE INDEX IF NOT EXISTS "shop_items_typ_aktiv_idx" ON "shop_items"("typ", "aktiv");
  CREATE INDEX IF NOT EXISTS "shop_items_rarity_idx" ON "shop_items"("rarity");

  -- 5. Create user_inventory_items table
  CREATE TABLE IF NOT EXISTS "user_inventory_items" (
    "id" TEXT NOT NULL,
    "userId" TEXT NOT NULL,
    "shopItemId" TEXT NOT NULL,
    "ausgeruestet" BOOLEAN NOT NULL DEFAULT false,
    "verbraucht" BOOLEAN NOT NULL DEFAULT false,
    "purchasedAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    "activatedAt" TIMESTAMP(3),

    CONSTRAINT "user_inventory_items_pkey" PRIMARY KEY ("id")
  );

  CREATE INDEX IF NOT EXISTS "user_inventory_items_userId_shopItemId_idx" ON "user_inventory_items"("userId", "shopItemId");
  CREATE INDEX IF NOT EXISTS "user_inventory_items_userId_ausgeruestet_idx" ON "user_inventory_items"("userId", "ausgeruestet");

  -- Foreign keys
  ALTER TABLE "user_inventory_items" ADD CONSTRAINT "user_inventory_items_userId_fkey"
    FOREIGN KEY ("userId") REFERENCES "users"("id") ON DELETE CASCADE ON UPDATE CASCADE;
  ALTER TABLE "user_inventory_items" ADD CONSTRAINT "user_inventory_items_shopItemId_fkey"
    FOREIGN KEY ("shopItemId") REFERENCES "shop_items"("id") ON DELETE CASCADE ON UPDATE CASCADE;

  COMMIT;
  ```

  DO $$ BEGIN
    ALTER TYPE "KalenderTyp" ADD VALUE IF NOT EXISTS 'FOKUSZEIT';
  EXCEPTION WHEN duplicate_object THEN NULL;
  END $$;

  BEGIN;
  -- rest of migration...
  COMMIT;

  **1. Shop Service (src/lib/gamification/shop-service.ts)**

  Create the service module with 4 exported functions. All DB operations that modify data use `prisma.$transaction`.

  Import: `prisma` from `@/lib/db`, `getLevelForXp` from `./game-profile-service`, `LEGENDARY_LEVEL_GATE` from `./types`, `createLogger` from `@/lib/logger`.

  ```typescript
  const log = createLogger("shop-service");
  ```

  **purchaseItem(userId: string, shopItemId: string)**
  - Wraps everything in `prisma.$transaction(async (tx) => { ... })`
  - Inside transaction:
    1. `tx.shopItem.findUnique({ where: { id: shopItemId } })` -- throw "Item nicht verfuegbar" if null or `!aktiv`
    2. `tx.userGameProfile.findUnique({ where: { userId } })` -- throw "Kein GameProfile" if null
    3. Level gate: if `item.rarity === "LEGENDARY"`, call `getLevelForXp(profile.xp)` and throw "Level 25 erforderlich" if level < LEGENDARY_LEVEL_GATE
    4. Balance check: if `profile.runen < item.preis`, throw "Nicht genuegend Runen"
    5. Atomic decrement: `tx.userGameProfile.update({ where: { userId }, data: { runen: { decrement: item.preis } } })`
    6. Create inventory: `tx.userInventoryItem.create({ data: { userId, shopItemId: item.id } })`
  - Return the created inventory item
  - Log: `log.info({ userId, shopItemId, preis: item.preis }, "Item purchased")`

  **equipItem(userId: string, inventoryItemId: string)**
  - Wraps in `prisma.$transaction(async (tx) => { ... })`
  - Inside transaction:
    1. `tx.userInventoryItem.findUnique({ where: { id: inventoryItemId }, include: { shopItem: true } })` -- throw "Nicht gefunden" if null or `invItem.userId !== userId`
    2. Throw "Perks koennen nicht ausgeruestet werden" if `invItem.shopItem.typ === "PERK"`
    3. Throw "Bereits verbraucht" if `invItem.verbraucht`
    4. Unequip all same-type items: `tx.userInventoryItem.updateMany({ where: { userId, ausgeruestet: true, shopItem: { typ: invItem.shopItem.typ } }, data: { ausgeruestet: false } })`
    5. Equip the new item: `tx.userInventoryItem.update({ where: { id: inventoryItemId }, data: { ausgeruestet: true } })`
  - Return updated inventory item with shopItem included

  **unequipItem(userId: string, inventoryItemId: string)**
  - Find the item: `prisma.userInventoryItem.findUnique({ where: { id: inventoryItemId } })`
  - Throw "Nicht gefunden" if null or wrong userId
  - Update: `prisma.userInventoryItem.update({ where: { id: inventoryItemId }, data: { ausgeruestet: false } })`
  - Return updated item

  **activatePerk(userId: string, inventoryItemId: string)**
  - Wraps in `prisma.$transaction(async (tx) => { ... })`
  - Inside transaction:
    1. `tx.userInventoryItem.findUnique({ where: { id: inventoryItemId }, include: { shopItem: true } })` -- throw "Nicht gefunden" if null or wrong userId
    2. Throw "Kein Perk" if `invItem.shopItem.typ !== "PERK"`
    3. Throw "Bereits verbraucht" if `invItem.verbraucht`
    4. Mark consumed: `tx.userInventoryItem.update({ where: { id: inventoryItemId }, data: { verbraucht: true, activatedAt: new Date() } })`
    5. Execute perk effect based on `(invItem.shopItem.metadata as Record<string, unknown>).perkType`:

  <files>src/lib/export/types.ts, src/lib/export/csv-export.ts, src/lib/export/xlsx-export.ts</files>
  <action>
Install ExcelJS: `npm install exceljs`

Create `src/lib/export/types.ts`:
```typescript
export type ExportFormat = "csv" | "xlsx";

export interface ExportColumn {
  key: string;       // field name in data object
  header: string;    // German column header
  width?: number;    // XLSX column width (characters)
  transform?: (value: any) => string | number; // optional value transformer
}

export interface ExportConfig {
  filename: string;  // without extension
  sheetName: string; // XLSX sheet name
  columns: ExportColumn[];
}
```

Create `src/lib/export/csv-export.ts`:
- Export `generateCsv(data: Record<string, any>[], config: ExportConfig): string`
- Use semicolon delimiter (German Excel default)
- UTF-8 BOM prefix (`\uFEFF`) for proper umlaut handling
- Header row from config.columns[].header
- Apply transform functions if defined
- Escape values containing semicolons or newlines with double-quote wrapping
- Return the CSV string

Create `src/lib/export/xlsx-export.ts`:
- Export `generateXlsx(data: Record<string, any>[], config: ExportConfig): Promise<Buffer>`
- Use ExcelJS streaming `WorkbookWriter` (per EXP-07 requirement) writing to a PassThrough stream
- Create worksheet with config.sheetName
- Set columns with header and width from config
- Header row: bold font, light blue fill (#DBEAFE), auto-filter enabled
- Add data rows, applying transform functions
- Commit workbook and return Buffer from stream
- This streaming approach handles large datasets without memory issues

  <files>src/app/api/akten/export/route.ts, src/app/api/kontakte/export/route.ts, src/app/api/finanzen/export/csv-xlsx/route.ts, src/components/export/export-button.tsx</files>
  <action>
**GET /api/akten/export?format=csv|xlsx**
- requireAuth + buildAkteAccessFilter
- Optional query params: status, sachgebiet, anwaltId (same filters as akten list)
- Query all matching Akte with: aktenzeichen, sachgebiet, status, kurzrubrum, wegen, anwalt.name, erstellt, geaendert
- Also include Beteiligte count and first few names via relation
- Columns: Aktenzeichen, Kurzrubrum, Sachgebiet, Status, Anwalt, Beteiligte, Erstellt, Geaendert
- If format=csv: return text/csv with Content-Disposition attachment
- If format=xlsx: return application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
- Filename: `akten-export-{YYYY-MM-DD}.{ext}`

**GET /api/kontakte/export?format=csv|xlsx**
- requireAuth
- Query Kontakt with: name/vorname/nachname (or firma), strasse, plz, ort, telefon, email, typ
- Check Prisma schema for exact Kontakt fields -- use actual field names
- Columns: Name, Firma, Strasse, PLZ, Ort, Telefon, E-Mail, Typ
- Same content-type and disposition pattern

**GET /api/finanzen/export/csv-xlsx?format=csv|xlsx&type=rechnungen|buchungen|zeiterfassung**
- requireAuth + requirePermission for finance access
- Based on `type` param:
  - `rechnungen`: Query Rechnung with nummer, rechnungsDatum, gesamtNetto, gesamtBrutto, status, akte.aktenzeichen
  - `buchungen`: Query AktenKontoBuchung with datum, betrag, beschreibung, typ, akte.aktenzeichen
  - `zeiterfassung`: Query Zeiterfassung with datum, dauer, beschreibung, user.name, akte.aktenzeichen
- Each type has its own column config
- Filename: `finanzen-{type}-{YYYY-MM-DD}.{ext}`

**ExportButton component** (`src/components/export/export-button.tsx`):
- "use client" component
- Props: `{ endpoint: string; formats?: ExportFormat[]; label?: string; params?: Record<string, string> }`
- Renders a split button or dropdown: click = default format (CSV), dropdown = choose format
- Use shadcn DropdownMenu + Button
- On click: fetch endpoint with format param, create blob URL, trigger download via hidden anchor click
- Show loading spinner during export
- Toast on error via sonner

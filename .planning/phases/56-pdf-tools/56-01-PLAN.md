  <name>Task 1: Create Stirling-PDF tools client library</name>
  <files>src/lib/pdf/stirling-pdf-tools.ts</files>
  <action>
Create `src/lib/pdf/stirling-pdf-tools.ts` with functions that call Stirling-PDF REST API endpoints. Follow the exact pattern from `src/lib/ocr/stirling-client.ts` (FormData with fileInput blob, fetch, return Buffer).

Stirling-PDF v1 API endpoints (all POST, multipart/form-data):

1. `mergePdfs(buffers: Buffer[], filenames: string[]): Promise<Buffer>`
   - Endpoint: `/api/v1/general/merge-pdfs`
   - Params: multiple `fileInput` entries (one per PDF)

2. `splitPdf(buffer: Buffer, pages: string): Promise<Buffer>`
   - Endpoint: `/api/v1/general/split-pdf`
   - Params: `fileInput` (PDF), `pageNumbers` (comma-separated page ranges like "1-3,5,7-9")

3. `rotatePdf(buffer: Buffer, angle: 90 | 180 | 270): Promise<Buffer>`
   - Endpoint: `/api/v1/general/rotate-pdf`
   - Params: `fileInput` (PDF), `angle` (number)

4. `compressPdf(buffer: Buffer, level: 1 | 2 | 3 | 4 | 5): Promise<Buffer>`
   - Endpoint: `/api/v1/general/compress-pdf`
   - Params: `fileInput` (PDF), `optimizeLevel` (1=low, 5=max compression)

5. `addWatermark(buffer: Buffer, text: string, fontSize?: number, rotation?: number, opacity?: number): Promise<Buffer>`
   - Endpoint: `/api/v1/security/add-watermark`
   - Params: `fileInput` (PDF), `watermarkText` (string), `fontSize` (default 30), `rotation` (default 45), `opacity` (default 0.5), `watermarkType` ("text")

6. `autoRedact(buffer: Buffer, searchText: string, useRegex: boolean, redactColor?: string): Promise<Buffer>`
   - Endpoint: `/api/v1/security/auto-redact`
   - Params: `fileInput` (PDF), `searchText` (string with PII patterns), `useRegex` (boolean), `redactColor` (hex, default "#000000")
   - For DSGVO: provide pre-built regex patterns for German PII (IBAN, phone, email, Steuernummer, Sozialversicherungsnummer)

7. Export a `DSGVO_PII_PATTERNS` constant with regex strings for common German PII types.

8. Export `PdfToolOperation` type union: `"merge" | "split" | "rotate" | "compress" | "watermark" | "redact"`

All functions: use `STIRLING_PDF_URL` env var (same as existing OCR client), throw descriptive errors on non-OK response.

  <action>
**Single-document endpoint: `POST /api/dokumente/[id]/pdf-tools`**

Create `src/app/api/dokumente/[id]/pdf-tools/route.ts`:
- Uses `requireAuth()` and `requireAkteAccess()` (follow pattern from `/api/dokumente/[id]/pdf/route.ts`)
- Accepts JSON body:
  ```typescript
  {
    operation: "split" | "rotate" | "compress" | "watermark" | "redact";
    // Operation-specific params:
    pages?: string;           // split: "1-3,5"
    angle?: 90 | 180 | 270;  // rotate
    level?: 1 | 2 | 3 | 4 | 5; // compress
    text?: string;            // watermark text
    fontSize?: number;        // watermark
    rotation?: number;        // watermark angle
    opacity?: number;         // watermark
    redactPatterns?: string[];// redact: custom patterns
    useDsgvo?: boolean;       // redact: use DSGVO_PII_PATTERNS
    saveAsNew?: boolean;      // true = create new Dokument, false = overwrite
  }
  ```
- Flow: load Dokument from DB, verify mimeType is "application/pdf", download from MinIO via `getFileStream`, call the matching stirling-pdf-tools function, save result back to MinIO.
- If `saveAsNew` (default true): create new Dokument record in same Akte with name suffix (e.g., "vertrag_komprimiert.pdf"), log audit event `DOKUMENT_HOCHGELADEN`.
- If `saveAsNew` is false: overwrite existing file in MinIO, update `groesse` on existing Dokument, log audit event `DOKUMENT_AKTUALISIERT`.
- Return `{ success: true, dokumentId, name, groesse }`.

**Multi-document merge endpoint: `POST /api/dokumente/pdf-tools/merge`**

Create `src/app/api/dokumente/pdf-tools/merge/route.ts`:
- Uses `requireAuth()`
- Accepts JSON body: `{ dokumentIds: string[], akteId: string, name?: string }`
- Validates all dokumentIds exist and belong to the given akteId (single DB query)
- Validates all are PDFs (mimeType check)
- Downloads all PDFs from MinIO, calls `mergePdfs()`, saves merged PDF as new Dokument in the Akte
- Default name: "Zusammengefuehrt_YYYY-MM-DD.pdf"
- Logs audit event with source document names
- Return `{ success: true, dokumentId, name, groesse }`

Both endpoints: use `try/catch` with specific error messages (German), return 400 for validation errors, 502 for Stirling-PDF errors.

**1. Create src/app/api/channels/[id]/messages/route.ts** with GET and POST:

**GET /api/channels/{id}/messages** -- Paginated message history:
- requireAuth()
- Verify user is a member of the channel (ChannelMember exists where channelId + userId) OR user is ADMIN
- For AKTE channels: if channel.typ === "AKTE" and channel.akteId is set, call requireAkteAccess(channel.akteId) as the authoritative access gate (per CONTEXT.md locked decision). ChannelMember rows are synced from RBAC but may be stale -- requireAkteAccess() is the single source of truth for AKTE channel access. If requireAkteAccess() returns an error, return that error immediately (403/404). This applies to both GET and POST handlers.
- Query messages: prisma.message.findMany with:
  - where: { channelId }
  - orderBy: { createdAt: "desc" }
  - cursor-based pagination: `?cursor={messageId}&limit=50` (default limit 50, max 100)
  - include: author (id, name, avatarUrl, isSystem), parent (id, author.name, body -- for reply-to preview), reactions (grouped)
- For reactions: include raw reactions, then transform client-side OR do a groupBy. Simplest: include all reactions, group by emoji in the response mapping: `{ emoji, count, userIds[] }`.
- Transform soft-deleted messages: if deletedAt is set, replace body with empty string (the "(bearbeitet)" / "Nachricht geloescht" label is a UI concern in Phase 32).
- Return: `{ messages: MessageListItem[], nextCursor: string | null }`

**2. Extend src/lib/queue/processors/helena-task.processor.ts** -- Post Helena response to channel:

Read the existing processor file to understand its structure. It processes HelenaTask jobs from BullMQ. After the task completes (ergebnis is set), check if the task has a channelId:

- Add a helper function `postHelenaResponseToChannel(channelId: string, helenaUserId: string, response: string): Promise<void>`:
  - Find or create a system user ID for Helena (query User where isSystem = true, cache the ID)
  - Create a Message in the channel with authorId = helenaUserId, body = response
  - Emit `message:new` via getSocketEmitter() to `channel:{channelId}` with isSystem: true flag
  - Wrap in try/catch, log warning on failure (non-fatal)

- In the main processing flow, after the task completes successfully and ergebnis is written:
  - Check: `if (task.channelId) { await postHelenaResponseToChannel(task.channelId, helenaSystemUserId, task.ergebnis); }`
  - The channelId comes from the HelenaTask record (added in 31-01 schema changes)

- For the system user lookup: use a module-level cached promise with idempotent upsert fallback to avoid repeated DB queries and runtime failure if no system user exists:
```typescript
let helenaUserIdPromise: Promise<string> | null = null;
async function getHelenaUserId(): Promise<string> {
  if (!helenaUserIdPromise) {
    helenaUserIdPromise = (async () => {
      // Try to find existing system user
      const existing = await prisma.user.findFirst({
        where: { isSystem: true },
        select: { id: true },
      });
      if (existing) return existing.id;
      // Fallback: create a system user for Helena (idempotent via upsert)
      const helena = await prisma.user.upsert({
        where: { email: "helena@system.local" },
        update: {},
        create: {
          email: "helena@system.local",
          name: "Helena",
          isSystem: true,
          role: "ADMIN",
          kanzleiId: (await prisma.kanzlei.findFirst({ select: { id: true } }))!.id,
        },
        select: { id: true },
      });
      return helena.id;
    })();
  }
  return helenaUserIdPromise;
}
```
This ensures @Helena channel responses never fail at runtime even if no isSystem user was previously seeded.

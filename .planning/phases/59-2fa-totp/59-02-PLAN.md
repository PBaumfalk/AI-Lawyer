  <action>
    Create 4 route handlers. All require an active session (return 401 if not authenticated). MANDANT role cannot enable 2FA (return 403).

    **src/app/api/auth/totp/setup/route.ts** — POST
    - Calls generateTotpSecret(user.email)
    - Stores the secret temporarily: `prisma.user.update({ where: { id }, data: { totpSecret: secret, totpEnabled: false } })`
    - Returns `{ qrCodeDataUrl, secret }` (secret shown for manual entry)
    - Does NOT enable 2FA yet — user must verify first

    **src/app/api/auth/totp/verify-setup/route.ts** — POST
    - Body: `{ code: string }` (6-digit TOTP)
    - Loads user.totpSecret from DB
    - Calls verifyTotpCode(secret, code) — return 400 `{ error: "Ungültiger Code" }` if false
    - On success: generate backup codes, update user:
      ```
      prisma.user.update({ data: { totpEnabled: true, totpVerifiedAt: new Date(), backupCodes: hashed } })
      ```
    - Returns `{ success: true, backupCodes: plain }` — plain backup codes shown ONCE

    **src/app/api/auth/totp/disable/route.ts** — POST
    - Body: `{ code: string }` (current TOTP code to confirm intent)
    - Verifies code against stored secret — return 400 if invalid
    - On success: `prisma.user.update({ data: { totpEnabled: false, totpSecret: null, backupCodes: [] } })`
    - Returns `{ success: true }`

    **src/app/api/auth/totp/backup-codes/route.ts** — POST (regenerate)
    - Body: `{ code: string }` (current TOTP to authorize regeneration)
    - Verifies code — return 400 if invalid
    - Generates new backup codes, stores hashed array
    - Returns `{ backupCodes: plain }` — shown ONCE

    All routes: import `auth` from "@/lib/auth", `prisma` from "@/lib/db", TOTP helpers from "@/lib/totp", and `logAuditEvent` from "@/lib/audit". Log audit events for TOTP_AKTIVIERT, TOTP_DEAKTIVIERT.

  <action>
    Create src/app/api/auth/totp/verify/route.ts — POST

    This route is called AFTER NextAuth password login succeeds but before the user is granted full session access. The login flow uses a pending-2FA session state stored in a short-lived httpOnly cookie.

    **Design:** Use a signed JWT cookie `totp_pending` that encodes the userId and expires in 10 minutes. Set by `auth.ts` when password auth succeeds and user has totpEnabled=true.

    For now, create this route to accept the pending token and a TOTP/backup code:
    - Body: `{ code: string, isBackupCode?: boolean }`
    - Read `totp_pending` cookie: `cookies().get('totp_pending')`
    - Verify JWT signature using `NEXTAUTH_SECRET` (use jsonwebtoken — already installed as @types/jsonwebtoken)
    - Load user from DB with totpSecret + backupCodes
    - If `isBackupCode=true`: call verifyBackupCode(code, user.backupCodes) — update DB with remainingCodes on success
    - Else: call verifyTotpCode(user.totpSecret, code)
    - On success:
      - Clear `totp_pending` cookie
      - Log TOTP_LOGIN_ERFOLG audit event
      - Return `{ success: true }` — client then calls NextAuth session refresh via `/api/auth/session`
    - On failure: return 401 `{ error: "Ungültiger Code" }` (do NOT clear cookie, allow retry)

    Note: The actual cookie-setting (when password check passes but TOTP pending) is handled in plan 03 when modifying auth.ts.

    Import: `import jwt from 'jsonwebtoken'` (package: jsonwebtoken, already in package.json as @types/jsonwebtoken — also add `jsonwebtoken` to dependencies if not present, currently it's only @types)

    Actually check: jsonwebtoken IS already in package.json (confirmed). Use it directly.

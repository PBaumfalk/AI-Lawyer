**Structure:**

2. **RECHTSGEBIET_TO_SACHGEBIET mapping** (constant, not exported):
   ```typescript
   const RECHTSGEBIET_TO_SACHGEBIET: Record<string, string[]> = {
     "Arbeitsrecht":     ["ARBEITSRECHT"],
     "Zivilrecht":       ["FAMILIENRECHT", "MIETRECHT", "ERBRECHT", "HANDELSRECHT", "INKASSO", "VERKEHRSRECHT"],
     "Verwaltungsrecht": ["VERWALTUNGSRECHT"],
     "Steuerrecht":      ["SONSTIGES"],
     "Sozialrecht":      ["SOZIALRECHT"],
     "Patentrecht":      ["SONSTIGES"],
     "Verfassungsrecht": [],  // Cross-cutting: no pre-filter, match all Akten
   };
   ```

3. **`scoreToSeverity(score: number): number`** helper (not exported):
   - >= 0.90 -> 9, >= 0.85 -> 8, >= 0.80 -> 7, >= 0.75 -> 6, else -> 5.

4. **`generateBriefing(params)`** helper (not exported):
   - Takes: `{ urteilContent: string, gericht: string, aktenzeichen: string, datum: string, akteSummaryText: string, kurzrubrum: string, sachgebiet: string, score: number }`.
   - Uses `getModel()` + `generateText()` from Vercel AI SDK.
   - System prompt: Helena persona, structured German format with 3 sections:
     1) "Was wurde entschieden?" (1-2 sentences about the Urteil)
     2) "Warum relevant fuer diese Akte?" (1-2 sentences connecting them)
     3) "Moegliche Auswirkungen" (1-2 sentences with actionable recommendations)
   - maxTokens: 500.
   - Wrap in try/catch — if LLM fails, return a fallback string: `"Automatische Analyse nicht verfuegbar. Bitte pruefen Sie das Urteil manuell."`.
   - Check and track token budget using `checkBudget()` + `trackTokenUsage()` if these exist, otherwise skip budget checks (use try/catch on import).

5. **`runNeuesUrteilCheck(): Promise<{ matched: number; alerts: number; errors: number }>`** (main export):
   - Check `getSettingTyped<boolean>("scanner.neues_urteil_enabled", true)` — return early with `{ matched: 0, alerts: 0, errors: 0 }` if disabled.
   - Read threshold: `getSettingTyped<number>("scanner.neues_urteil_threshold", 0.72)`.
   - Find newly ingested Urteile: query urteil_chunks where `ingestedAt >= NOW() - INTERVAL '1 day'` AND `piiFiltered = true` AND `embedding IS NOT NULL`. Use `prisma.$queryRaw` for the vector null check.
   - For each new Urteil:
     a) Determine Sachgebiet filter from RECHTSGEBIET_TO_SACHGEBIET. If rechtsgebiet is null or "Verfassungsrecht" (empty array), skip the sachgebiet WHERE clause (match all Akten).
     b) Run pgvector cosine similarity query against akten table:
        ```sql
        SELECT a.id AS "akteId", a.kurzrubrum, a."anwaltId", a."sachbearbeiterId", a.sachgebiet,
               1 - (a."summaryEmbedding" <=> uc.embedding) AS score
        FROM akten a
        INNER JOIN urteil_chunks uc ON uc.id = ${urteilChunkId}
        WHERE a.status = 'OFFEN'
          AND a."summaryEmbedding" IS NOT NULL
          ${sachgebietFilter ? `AND a.sachgebiet = ANY(ARRAY[${sachgebietValues}]::text[])` : ''}
          AND 1 - (a."summaryEmbedding" <=> uc.embedding) >= ${threshold}
        ORDER BY score DESC
        ```
        Use `prisma.$queryRaw` with Prisma.sql tagged template for safe parameterization. For the sachgebiet filter, use `Prisma.sql` conditional composition (not string interpolation).
     c) For each match above threshold:
        - Resolve alert recipients via `resolveAlertRecipients({ anwaltId, sachbearbeiterId })`.
        - Call `assembleAkteSummaryText()` — need to fetch the full Akte with includes for this. Cache this per Akte ID to avoid re-fetching if same Akte matches multiple Urteile.
        - Generate briefing via `generateBriefing()`.
        - For each recipient: call `createScannerAlert()` with:
          - `typ: "NEUES_URTEIL"` (already in HelenaAlertTyp enum)
          - `titel: "Relevantes Urteil fuer [kurzrubrum]"` (per user decision)
          - `inhalt: briefing` (the LLM-generated briefing text)
          - `severity: scoreToSeverity(score)`
          - `meta: { urteilChunkId, gericht, aktenzeichen, datum, sourceUrl, score, rechtsgebiet }`
        - **IMPORTANT dedup fix:** The existing `createScannerAlert()` uses 24h dedup on `akteId + typ`. For NEUES_URTEIL, we need per-Urteil dedup. BEFORE calling createScannerAlert, check:
          ```typescript
          const existingAlert = await prisma.helenaAlert.findFirst({
            where: {
              akteId: match.akteId,
              typ: "NEUES_URTEIL",
              meta: { path: ["urteilChunkId"], equals: urteilChunkId },
            },
          });
          if (existingAlert) { /* skip — already alerted for this Urteil-Akte pair */ continue; }
          ```
          This replaces the time-based dedup for NEUES_URTEIL with a permanent per-Urteil dedup (since NEUES_URTEIL alerts don't auto-resolve per user decision).
     d) Wrap each Urteil processing in try/catch (per-Urteil error isolation).
   - Return `{ matched, alerts, errors }` counts.

## Cross-Phase Integration

### Verified Integration Points (7/8)

1. **Phase 33 -> 34** (Profile -> Dashboard): `/api/gamification/dashboard` calls `getOrCreateGameProfile()` + `evaluateQuestCondition()` from Phase 33 services. Opt-in gate consistent.
2. **Phase 33 -> 35** (Quest -> Boss): Shared `gamificationQueue`; `awardRewards()` for boss rewards; nightly safety net triggers `checkAndSpawnBoss`.
3. **Phase 34 + 35 -> Dashboard**: `QuestWidget` and `BossfightBanner` both render without conflict; absent-until-loaded pattern.
4. **Phase 36** (Independent): `AkteDetailClient` and `OcrRecoveryBanner` self-contained. Recovery APIs wire into embedding pipeline.
5. **Phase 37 -> 33**: Class-filtered quests via `Quest.klasse` + OR filter in `quest-service.ts` and dashboard API. WeeklySnapshot cron registered.
6. **Phase 38 -> 33**: Audit sampling, Runen cap, qualified completion check all wired into `quest-service.ts` reward flow.
7. **Socket.IO**: `SocketProvider` wraps dashboard layout; `setupRooms` auto-joins `kanzlei:{id}`. All 5 boss event shapes match consumer contracts (post Plan 35-03 fix). Audit event `gamification:audit-needed` wired via `GamificationAuditListener` in layout.

### Critical Integration Break (1/8)

**`/api/kalender/[id]/erledigt` -> `enqueueQuestCheck` MISSING**

The primary UI path for marking WV/Frist as erledigt (used by `kalender-liste.tsx`, `kalender-tab.tsx`, `tagesuebersicht.tsx`) does not call `enqueueQuestCheck`. Boss damage IS correctly enqueued. Only the quest evaluation hook is missing.

- **Impact:** Quest completions for WV/Frist work delayed until nightly safety-net cron (23:55). No real-time quest feedback.
- **Fix:** Add `enqueueQuestCheck(userId)` after the boss-damage block in `src/app/api/kalender/[id]/erledigt/route.ts` (same pattern as `/api/kalender/[id]/route.ts` line 136 and `/api/tickets/[id]/route.ts` line 118).
- **Affected requirements:** QUEST-03, ABUSE-01, ABUSE-02, ABUSE-03, ABUSE-04

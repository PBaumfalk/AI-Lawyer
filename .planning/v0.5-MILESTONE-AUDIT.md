## Remaining Gap: Dokumente Page Navigation

**Severity:** Medium (functional but undiscoverable)

The dokumente page (`/portal/akten/[id]/dokumente`) exists and works correctly — API routes wired, download works, upload works. However, there is no navigation link to reach it from within the portal UI. The sidebar "Dokumente" link was removed (user decision in Phase 49 discuss). No alternative link was added on the akte detail page.

**Workaround:** MSG-05 email deep link correctly resolves to the dokumente page.

**Fix:** Add a "Dokumente" card/link on the akte detail page (`/portal/akten/[id]`).

**Affected requirements:** DOC-02 (list), DOC-03 (download), DOC-04 (upload) — all implemented but navigation-orphaned.

## Architectural Notes

- Portal uses dual route groups: `(portal)` (guarded) and `(portal-public)` (public) with shared `/portal/*` URL namespace
- Both groups use `portal/` subdirectory for URL segment: `(portal)/portal/dashboard` → `/portal/dashboard`
- 10s polling for portal messages (no Socket.IO in portal MVP — sufficient for Mandant use case)
- Orphaned API routes kept for potential external consumers (mobile, integrations)

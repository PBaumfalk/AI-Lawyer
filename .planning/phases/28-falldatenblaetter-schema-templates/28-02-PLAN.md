- Query params: `?status=` (optional filter), `?sachgebiet=` (optional filter), `?eigene=true` (only own templates)

  ```typescript
  where: {
    OR: [
      { status: { in: ["GENEHMIGT", "STANDARD"] } },
      { erstelltVonId: userId },
    ],
    // Apply additional filters if provided:
    ...(statusFilter ? { status: statusFilter } : {}),
    ...(sachgebietFilter ? { sachgebiet: sachgebietFilter } : {}),
  }
  ```

Create 4 route files for template detail operations and workflow transitions.

**A) `src/app/api/falldaten-templates/[id]/route.ts`** — GET, PATCH, DELETE:

**GET** — Get template detail:
- Auth: any authenticated user
- Visibility: GENEHMIGT/STANDARD visible to all; other statuses only to creator or ADMIN
- Include erstelltVon (name), geprueftVon (name)
- Return 404 if not found or not visible

**PATCH** — Edit template:
- Auth: creator only (erstelltVonId === session.user.id)
- Only allowed when status is ENTWURF or ABGELEHNT (return 403 otherwise with message "Template kann nur im Entwurf oder nach Ablehnung bearbeitet werden")
- STANDARD templates cannot be edited (return 403 "Standard-Templates koennen nicht bearbeitet werden")
- Parse body with `updateTemplateSchema.safeParse(body)` — partial update
- If status was ABGELEHNT, keep status as ABGELEHNT (user must explicitly resubmit via einreichen)
- Return updated template

**DELETE** — Delete template:
- Auth: creator only
- Only allowed when status is ENTWURF (return 403 otherwise "Nur Entwuerfe koennen geloescht werden")
- STANDARD templates cannot be deleted (return 403)
- Soft check: if any Akte has falldatenTemplateId pointing to this template, return 409 "Template wird von Akten verwendet"
- Return 204 on success

Param extraction: `{ params }: { params: { id: string } }` — use `params.id` for Prisma lookup.

**B) `src/app/api/falldaten-templates/[id]/einreichen/route.ts`** — POST submit for review:
- Auth: creator only (erstelltVonId === session.user.id)
- Only from ENTWURF or ABGELEHNT status (return 400 otherwise)
- STANDARD templates cannot be submitted (return 403)
- Update status to EINGEREICHT, clear ablehnungsgrund (if resubmitting after rejection)
- Return updated template with 200

**C) `src/app/api/falldaten-templates/[id]/genehmigen/route.ts`** — POST approve:
- Auth: ADMIN only
- Only from EINGEREICHT status (return 400 otherwise)
- Self-approval check per CONTEXT decision: If only 1 active ADMIN exists, Admin CAN approve their own. If multiple active ADMINs exist, cannot approve own (return 403 "4-Augen-Prinzip: Eigene Templates koennen nicht selbst genehmigt werden")
- Update: status = GENEHMIGT, geprueftVonId = admin user id, geprueftAt = now()
- Create Notification for erstelltVonId:
  ```typescript
  await prisma.notification.create({
    data: {
      userId: template.erstelltVonId,
      type: "template:approved",
      title: "Falldaten-Template genehmigt",
      message: `Ihr Template "${template.name}" wurde genehmigt und ist jetzt fuer alle Nutzer verfuegbar.`,
      data: { templateId: template.id },
    },
  });
  ```
- Push real-time notification via Socket.IO emitter (import getSocketEmitter from `@/lib/socket/emitter`):
  ```typescript
  const emitter = getSocketEmitter();
  if (emitter) {
    emitter.to(`user:${template.erstelltVonId}`).emit("notification", { type: "template:approved", templateId: template.id });
  }
  ```
- Return updated template with 200

**D) `src/app/api/falldaten-templates/[id]/ablehnen/route.ts`** — POST reject:
- Auth: ADMIN only
- Only from EINGEREICHT status (return 400 otherwise)
- Parse body with `rejectTemplateSchema.safeParse(body)` — ablehnungsgrund required
- Update: status = ABGELEHNT, geprueftVonId = admin user id, geprueftAt = now(), ablehnungsgrund from body
- Create Notification for erstelltVonId:
  ```typescript
  await prisma.notification.create({
    data: {
      userId: template.erstelltVonId,
      type: "template:rejected",
      title: "Falldaten-Template abgelehnt",
      message: `Ihr Template "${template.name}" wurde abgelehnt. Grund: ${ablehnungsgrund}`,
      data: { templateId: template.id, ablehnungsgrund },
    },
  });
  ```
- Push Socket.IO notification (same pattern as approve)
- Return updated template with 200

All routes: import `NextRequest`, `NextResponse` from `next/server`, `auth` from `@/lib/auth`, `prisma` from `@/lib/db`. Use consistent error response pattern: `{ error: string }` with appropriate HTTP status codes. Wrap DB operations in try/catch with 500 fallback.

<task type="auto">
  <name>Task 1: Create PDF report generator and endpoint</name>
  <files>src/lib/bi/pdf-report.ts, src/app/api/bi/export/pdf/route.ts</files>
  <action>
Install jsPDF: `npm install jspdf`

Create `src/lib/bi/pdf-report.ts`:
- Export `generateBiPdfReport(tiles: KpiTile[], trends: TrendSeries[], zeitraum: { von: string; bis: string }, kanzleiName?: string): Buffer`
- Use jsPDF to generate A4 portrait document:
  1. **Briefkopf (header)**: Kanzlei name (from settings or param, default "Kanzlei") in bold 18pt at top, followed by a horizontal line. Below: "BI-Report" subtitle and date range in smaller text.
  2. **KPI Section**: "Kennzahlen" heading. Render a table with columns: KPI, Wert, Vorperiode, Veraenderung (%). Group rows by domain with domain headers (Akten, Finanzen, Fristen, Helena). Format values with units (EUR, %, count).
  3. **Trend Section**: "Trends" heading. For each trend series, render a simple data table with Month | Value columns (charts cannot easily be rendered in jsPDF without canvas -- tables are the pragmatic approach for server-side PDF).
  4. **Footer**: Page numbers, generation timestamp
- Handle page breaks when content exceeds page height
- Return the PDF as a Buffer via `doc.output("arraybuffer")`

Create `src/app/api/bi/export/pdf/route.ts`:
- GET handler with requireAuth
- Parse BiFilterParams from query params
- Build accessFilter from session
- Fetch all KPI tiles and trend data using the same query functions from kpi-queries.ts (via Promise.all)
- Optionally fetch Kanzlei name from settings (check if there's a settings table/API -- if not, use default)
- Call generateBiPdfReport
- Return Response with content-type application/pdf, Content-Disposition: attachment; filename="bi-report-{YYYY-MM-DD}.pdf"
  </action>
  <verify>npx tsc --noEmit src/lib/bi/pdf-report.ts src/app/api/bi/export/pdf/route.ts 2>&1 | head -20</verify>
  <done>PDF report endpoint generates A4 document with Kanzlei-Briefkopf, KPI table grouped by domain with deltas, and trend data tables. Returns downloadable PDF.</done>
</task>

Create `src/lib/bi/xlsx-report.ts`:
- Export `generateBiXlsxReport(tiles: KpiTile[], trends: TrendSeries[], zeitraum: { von: string; bis: string }): Promise<Buffer>`
- Use ExcelJS streaming WorkbookWriter (already installed from Plan 02):
  1. **Sheet "Kennzahlen"**: Columns: Domain, KPI, Wert, Vorperiode, Delta (%). Add all tiles as rows. Format: header row bold + light blue fill, auto-filter, column widths set. Group rows by domain.
  2. **Sheet "Akten-Neuzugang"**: Columns: Monat, Anzahl. Data from akten-neuzugang trend series.
  3. **Sheet "Umsatz"**: Columns: Monat, Betrag (EUR). Data from umsatz-monat trend series.
  4. **Sheet "Fristen-Compliance"**: Columns: Monat, Rate (%). Data from fristen-compliance trend series.
- Each sheet: formatted headers, appropriate number formats, auto-filter
- Return Buffer

Create `src/app/api/bi/export/xlsx/route.ts`:
- GET handler with requireAuth
- Parse BiFilterParams, build accessFilter
- Fetch KPI tiles + trend data
- Call generateBiXlsxReport
- Return Response with content-type application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
- Content-Disposition: attachment; filename="bi-report-{YYYY-MM-DD}.xlsx"

  - "Excel-Report" button with FileSpreadsheet icon -- triggers GET /api/bi/export/xlsx with current filters
- On click: fetch with current filter params as query string, create blob, trigger download
- Show loading state on button during download
- Use existing shadcn Button styling

**Integration**: Update `src/app/(dashboard)/bi/page.tsx` to include the ExportBar component, passing current filters. Place it in the header area next to the title or filters.

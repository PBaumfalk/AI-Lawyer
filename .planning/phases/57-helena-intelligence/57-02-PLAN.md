Create `src/lib/helena/case-summary.ts`:

1. Export types:
   ```typescript
   export interface TimelineEvent {
     datum: string;       // ISO date string
     titel: string;       // short title
     beschreibung: string; // 1-2 sentence description
     typ: "DOKUMENT" | "FRIST" | "TERMIN" | "KOMMUNIKATION" | "ENTSCHEIDUNG" | "SONSTIGES";
     quellDokument?: string; // document name if from a doc
   }

   export interface KeyFact {
     label: string;   // e.g. "Streitwert", "Gegner", "Naechste Frist"
     value: string;   // the fact value
     kategorie: "PARTEI" | "VERFAHREN" | "FINANZEN" | "FRIST" | "SONSTIGES";
   }

   export interface CaseSummary {
     zusammenfassung: string;      // 3-5 sentence case summary
     timeline: TimelineEvent[];     // chronological events
     keyFacts: KeyFact[];           // structured key facts
     generatedAt: string;           // ISO timestamp
   }
   ```

2. Export async function `generateCaseSummary(options)`:
   - Accept: `{ prisma, akteId, userId: string }`
   - Load Akte with: beteiligte (include kontakt), dokumente (recent 15, with ocrText LEFT 2000 chars via raw query), kalenderEintraege (all), falldaten, sachgebiet, kurzrubrum, wegen, status, gegenstandswert, aktenzeichen
   - Build context string from: Akte metadata, beteiligte names+roles, document names+OCR snippets, Fristen/Termine, existing falldaten
   - Call `generateObject` with Zod schema matching CaseSummary structure:
     - zusammenfassung: z.string()
     - timeline: z.array(z.object({...}))
     - keyFacts: z.array(z.object({...}))
   - System prompt (German): "Du bist Helena, KI-Assistentin einer Kanzlei. Erstelle eine strukturierte Fallzusammenfassung. Extrahiere: 1) Eine kurze Zusammenfassung (3-5 Saetze). 2) Eine chronologische Timeline der wichtigsten Ereignisse (Dokumente, Fristen, Termine, Entscheidungen). 3) Key Facts (Parteien, Verfahrensstand, Streitwert, naechste Fristen). Basiere alles auf den vorliegenden Dokumenten und Aktendaten. Erfinde keine Fakten."
   - Add generatedAt timestamp
   - Track token usage via trackTokenUsage
   - Return CaseSummary

3. Create `src/app/api/akten/[id]/zusammenfassung/route.ts`:
   - GET handler with requireAuth + requireAkteAccess
   - Call generateCaseSummary
   - Return JSON: `{ summary: CaseSummary }`
   - Cache consideration: no caching for now (generation is on-demand), user triggers refresh manually
   - Handle errors with try/catch, return 500 on failure

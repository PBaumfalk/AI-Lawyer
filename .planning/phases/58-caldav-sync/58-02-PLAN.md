1. Create src/lib/caldav/ical-builder.ts:
- kalenderEintragToVEvent(eintrag: KalenderEintrag): string
  - Build a VCALENDAR/VEVENT iCalendar string from KalenderEintrag fields
  - Map: titel -> SUMMARY, beschreibung -> DESCRIPTION, datum -> DTSTART, datumBis -> DTEND, ganztaegig -> VALUE=DATE format
  - For Fristen: prepend "[FRIST] " to SUMMARY, set TRANSP:TRANSPARENT (non-blocking)
  - For Termine: normal SUMMARY, set TRANSP:OPAQUE
  - Generate deterministic UID from eintrag.id: `${eintrag.id}@ai-lawyer.local`
  - Include DTSTAMP and LAST-MODIFIED from eintrag.updatedAt
- vEventToCalDavEvent(icalString: string): CalDavEvent
  - Parse a VEVENT string to extract CalDavEvent fields
  - Handle both DATE and DATE-TIME formats for all-day vs timed events
  - Use simple regex parsing (no ical.js dependency -- events are simple, no RRULE per requirements)

2. Create src/lib/caldav/sync-engine.ts:
- syncCalDavKonto(kontoId: string): Promise<{ pushed: number, pulled: number, deleted: number, errors: string[] }>
  - Load CalDavKonto from DB with decrypted credentials
  - Create tsdav client via createCalDavClient
  - Check calendar CTag: if unchanged from stored ctag, skip sync (no changes)

  Phase A -- PUSH Fristen (read-only export, richtung="PUSH"):
  - Query all KalenderEintrag where typ=FRIST AND verantwortlichId=konto.userId AND erledigt=false
  - For each Frist:
    - Check CalDavSyncMapping for existing mapping
    - If no mapping: createEvent on CalDAV, create mapping with richtung=PUSH
    - If mapping exists: compare eintrag.updatedAt with mapping.letzterSync
      - If eintrag is newer: updateEvent on CalDAV with stored etag, update mapping
      - If same: skip
  - For Frist mappings where KalenderEintrag is now erledigt=true or deleted: deleteEvent on CalDAV, delete mapping

  Phase B -- BIDI Termine (bidirectional, richtung="BIDI"):
  - Query all KalenderEintrag where typ=TERMIN AND verantwortlichId=konto.userId
  - Fetch all events from CalDAV calendar

  Push local changes to CalDAV:
  - For each local Termin:
    - Check CalDavSyncMapping
    - No mapping: createEvent, create mapping with richtung=BIDI
    - Mapping exists, local newer (updatedAt > mapping.letzterSync): updateEvent
    - Mapping exists, remote etag changed: CONFLICT -- remote wins (safer for external calendar authority)

  Pull remote changes to local:
  - For each CalDAV event not in mappings (new external event):
    - Only if UID does not match our pattern (*@ai-lawyer.local) -- skip our own pushed events
    - Create mapping with richtung=PULL, kalenderEintragId=null (will be displayed separately, see Plan 03)
  - For each mapping with richtung=PULL or BIDI: check if remote etag changed
    - If etag changed and richtung=BIDI: update local KalenderEintrag from remote event data
    - If remote event deleted (not in fetch results): if richtung=BIDI, mark local Termin erledigt; if PULL, delete mapping

  Phase C -- Update sync state:
  - Update CalDavKonto.ctag, letzterSync, syncStatus=VERBUNDEN
  - On any error: append to fehlerLog (keep last 5), set syncStatus=FEHLER if critical

  Use createLogger("caldav-sync") for detailed logging. Wrap entire sync in try/catch to never crash the worker.

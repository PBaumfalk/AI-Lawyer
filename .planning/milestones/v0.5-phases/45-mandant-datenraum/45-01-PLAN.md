  **2. Portal access library (`src/lib/portal-access.ts`):**

  Create a portal-specific access control library modeled on the `requireAkteAccess` pattern from `src/lib/rbac.ts`.

  Export two functions:

  a) `getMandantAkten(userId: string)` -- Returns all Akten where the User's linked Kontakt (via `user.kontaktId`) is a Beteiligter with rolle=MANDANT. Query chain: User.kontaktId -> Beteiligter(kontaktId, rolle=MANDANT) -> Akte. Return: array of Akte summaries with { id, aktenzeichen, kurzrubrum, sachgebiet, status, naechsteSchritte }.

  b) `requireMandantAkteAccess(akteId: string, userId: string)` -- Verifies the given user has MANDANT access to the specific Akte via the same Kontakt -> Beteiligter chain. Returns `{ akte, error? }` pattern matching rbac.ts conventions. Returns 404 (not 403) if unauthorized to hide Akte existence.

  Both functions enforce server-side isolation at query level -- no client-side filtering. Import `prisma` from `@/lib/db`.

  NOTE: Phase 43 will have added `kontaktId` to the User model (linking MANDANT users to their Kontakt record). Use `user.kontaktId` as the starting point for the access chain. If Phase 43 used a different FK direction (e.g., `kontakt.portalUserId`), adjust accordingly -- check the actual schema after Phase 43/44 execution.

  - Return JSON with Akte overview data (SACH-03)
  - IMPORTANT: Do NOT expose internal fields like anwaltId, sachbearbeiterId, kanzleiId, gegenstandswert, falldaten -- only Mandant-relevant info

  - Call `requireAuth()`, verify MANDANT role
  - Call `requireMandantAkteAccess(akteId, session.user.id)` -- returns 404 if unauthorized
  - Query `prisma.aktenActivity.findMany()` with filters:
    - `akteId: akteId`
    - `mandantSichtbar: true` (CRITICAL: only show approved events)
    - Order by `createdAt: "desc"` (newest first)
  - Support cursor-based pagination (same pattern as existing `/api/akten/[id]/feed`): `take` (default 20, max 100), `cursor` query param
  - Return `{ items, nextCursor, hasMore }` (SACH-01)
  - Each item includes: id, typ, titel, inhalt (truncated), createdAt -- NO user info, NO meta details

  - Return updated Akte with naechsteSchritte field (SACH-02)

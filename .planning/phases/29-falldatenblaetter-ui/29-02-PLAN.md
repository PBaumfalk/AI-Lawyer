  <action>
  Create `src/components/akten/falldaten-tab.tsx` -- a "use client" component that manages template resolution, auto-assignment, and renders FalldatenForm.

  **Props interface:**
  ```typescript
  interface FalldatenTabProps {
    akteId: string;
    sachgebiet: string;
    initialFalldaten: Record<string, any> | null;
    falldatenTemplateId: string | null;
    onCompletenessChange: (completeness: { percent: number; filled: number; total: number }) => void;
    onDirtyChange: (dirty: boolean) => void;
  }
  ```

  **Component state:**
  - `template`: The resolved template object (`{ id, name, beschreibung, schema }` or null)
  - `loading`: Boolean for initial template fetch
  - `alternatives`: Array of alternative templates for "Template wechseln"
  - `showSwitchDialog`: Boolean for template switch AlertDialog
  - `pendingSwitchTemplate`: The template to switch to (held until user confirms)

  **Template resolution flow (useEffect on mount):**
  1. If `falldatenTemplateId` is set: fetch the template via `GET /api/falldaten-templates/${falldatenTemplateId}` and set it in state. Also fetch alternatives.
  2. If `falldatenTemplateId` is null: auto-assign by fetching `GET /api/falldaten-templates?sachgebiet=${sachgebiet}&status=STANDARD`. If a STANDARD template is found, PATCH the Akte with `{ falldatenTemplateId: template.id }` to persist the assignment, then set the template in state. If no STANDARD found, try `GET /api/falldaten-templates?sachgebiet=${sachgebiet}&status=GENEHMIGT` for approved community templates. If still nothing found, show empty state (template stays null).
  3. After resolving, fetch alternatives: `GET /api/falldaten-templates?sachgebiet=${sachgebiet}` (fetch both STANDARD and GENEHMIGT), filter out the currently assigned template.

  **Loading state:** Show a skeleton with 3 glass-card-shaped shimmer divs while loading.

  **Empty state (no template found):**
  Show a centered card with `FileSpreadsheet` icon, message "Fuer dieses Sachgebiet ist kein Standardtemplate verfuegbar", and a brief explanation. If there are ANY GENEHMIGT templates for the sachgebiet, show them as selectable options. Otherwise, suggest creating a custom template.

  **Template switch flow:**
  1. Show "Template wechseln" button in the header area (next to template name), only when `alternatives.length > 0`.
  2. Clicking opens an AlertDialog with:
     - Title: "Template wechseln"
     - Description: "Vorhandene Daten bleiben erhalten. Felder, die im neuen Template nicht vorhanden sind, werden nicht angezeigt, aber gespeichert."
     - A list of alternative templates (radio buttons or clickable cards with name + description)
     - Cancel + "Wechseln" action buttons
  3. On confirm: PATCH `/api/akten/${akteId}` with `{ falldatenTemplateId: newTemplate.id }`, update template state, re-filter alternatives.

  **Rendering (when template is loaded):**
  ```tsx
  <FalldatenForm
    akteId={akteId}
    schema={{ label: template.name, beschreibung: template.beschreibung, felder: template.schema.felder }}
    initialData={initialFalldaten}
    onCompletenessChange={onCompletenessChange}
    onDirtyChange={onDirtyChange}
  />
  ```

  **Imports:**
  - `FalldatenForm` from `./falldaten-form`
  - `AlertDialog, AlertDialogContent, AlertDialogHeader, AlertDialogTitle, AlertDialogDescription, AlertDialogFooter, AlertDialogAction, AlertDialogCancel` from `@/components/ui/alert-dialog`
  - `Button` from `@/components/ui/button`
  - `FileSpreadsheet, Loader2, RefreshCw` from `lucide-react`
  - `toast` from `sonner`
  - `useState, useEffect, useCallback` from `react`

  **Glass UI styling:** Follow the existing FalldatenForm pattern for glass cards: `bg-white/50 dark:bg-white/[0.05] backdrop-blur-md rounded-xl border border-white/20 dark:border-white/[0.08]`.

     ```typescript
     const handleTabChange = useCallback((newTab: string) => {
       // If leaving Falldaten tab with unsaved changes, show warning
       if (activeTab === "falldaten" && falldatenDirty && newTab !== "falldaten") {
         setPendingTab(newTab);
         setShowUnsavedDialog(true);
         return;
       }
       setActiveTab(newTab);
     }, [activeTab, falldatenDirty]);
     ```

  4. **Add browser beforeunload guard:**
     ```typescript
     useEffect(() => {
       const handler = (e: BeforeUnloadEvent) => {
         if (falldatenDirty) { e.preventDefault(); }

       window.addEventListener("beforeunload", handler);
       return () => window.removeEventListener("beforeunload", handler);
     }, [falldatenDirty]);
     ```
